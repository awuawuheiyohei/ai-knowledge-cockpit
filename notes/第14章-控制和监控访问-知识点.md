# 第14章 控制和监控访问

### 14.1 比较访问控制模型

#### 核心概念：权限、权利与特权
| 概念 | 英文 | 定义 |
|---|---|---|
| **权限** | Permissions | 授予对象的访问权限，决定可以对其执行的操作。 |
| **权利** | Rights | 对对象采取行动的能力。 |
| **特权** | Privileges | 提升的权利和权限的组合。 |

#### 核心原则
*   **隐式拒绝**：除非明确授予主体访问权限，否则拒绝对对象的访问。
*   **需要知道**：仅授予主体执行其工作任务和职能所必需知道的访问权限。
*   **最小特权**：仅授予主体执行其工作任务和职能所需的特权。
*   **职责和责任分离**：将敏感功能拆分为由两名或多名员工执行的任务，建立制衡系统以防止欺诈和错误。

> 💡 **记忆类比**：Need to Know 管"知不知道"（信息可见性），Least Privilege 管"能不能做"（操作权限），Separation of Duties 管"一人能不能全做"（防勾结制衡）。

#### 访问控制矩阵与能力表
| 概念 | 特性 | 描述 |
|---|---|---|
| **访问控制矩阵** | 集中式 | 包含主体、客体以及分配的权限、权利和特权的集中表格。 |
| **能力表** | 分散式/分布式 | 以主体为中心，使用令牌或密钥识别分配给主体的权限，识别主体可访问的对象。 |

#### 接口与依赖控制
*   **受限接口**：应用程序根据用户特权限制用户可以执行或查看的操作。
*   **基于内容的控制**：根据对象内的内容限制对数据的访问。
*   **基于上下文的控制**：要求在授予用户访问权限之前进行特定的活动。

> ⚠️ **考试陷阱**：Content-Dependent（基于内容）看的是"数据本身包含什么"；Context-Dependent（基于上下文）看的是"前置条件/当前状态是什么"。

#### 访问控制模型
| 模型 | 英文缩写 | 关键特征 |
|---|---|---|
| **自主访问控制** | DAC | 每个对象都有所有者，所有者可授予或拒绝其他主体的访问。 |
| **基于角色的访问控制** | RBAC | 使用角色或组，特权分配给角色而非直接分配给用户，角色通常由工作职能确定。 |
| **基于任务的访问控制** | TBAC | 将用户分配给一系列任务，通过分配的任务而非身份或角色控制访问。 |
| **基于规则的访问控制** | Rule-Based | 使用规则、限制或过滤器，具有适用于所有主体的**全局规则**。 |
| **基于属性的访问控制** | ABAC | 使用包含规则的多个属性的策略，属性可以是用户、网络和设备的任何特征。 |
| **强制访问控制** | MAC | 依赖分类标签，对象和主体都有预定义标签，系统根据标签确定访问权限。 |
| **基于风险的访问控制** | Risk-Based | 评估风险和环境后授予访问权限，使用嵌入在软件代码中的策略做出动态决策。 |

> ⚠️ **常见混淆点**：RBAC 可以是 Role-Based 也可以是 Rule-Based，注意题目中的全称。Rule-Based 强调"全局规则"，ABAC 强调"多属性组合"。

### 14.2 实现认证系统

#### SAML 2.0 体系
*   **SAML 2.0**：基于 XML 的开放标准，用于在联邦组织之间交换身份验证和授权 (AA) 信息，为浏览器访问提供 SSO 功能。
*   **主体**：尝试访问服务提供商站点并需要进行身份验证的用户。
*   **服务提供商 (SP / Relying Party)**：提供具体服务的实体，依赖身份提供商验证用户凭证。
*   **身份提供商**：持有用户验证和授权信息的第三方，负责验证凭据并返回 SAML 断言。

#### OAuth 2.0 与 OpenID Connect (OIDC)
*   **OAuth 2.0**：授权框架，用于共享账户资源访问权限。交换 API 消息并使用令牌，**本身不提供身份验证协议**，不暴露用户凭据。
*   **OpenID Connect (OIDC)**：建立在 OAuth 2.0 之上的**身份验证层**。使用 JSON Web Token (JWT / ID 令牌)，同时提供身份验证和授权功能。

> ⚠️ **考试陷阱**：OAuth 2.0 仅用于**授权**（不包含认证），OIDC 在 OAuth 2.0 之上增加了**认证**层。

#### Kerberos 体系
*   **Kerberos**：基于票据的身份验证系统，使用 AES 对称密钥密码术，提供 SSO，防窃听和重放攻击。
*   **密钥分发中心 (KDC)**：受信任第三方，维护所有成员私密密钥，生成加密会话密钥。
*   **身份验证服务器 (AS)**：托管 KDC 功能，验证票据真实性和时效性。
*   **票据**：加密消息，证明主体有权访问对象/服务，具有特定生存期。
*   **票据授予票据 (TGT)**：证明主体已通过 KDC 验证并被授权请求其他票据。包含对称密钥、过期时间及用户 IP 等。
*   **Kerberos 领域**：由 Kerberos 控制的逻辑区域，领域内主体可向 KDC 请求/接收票据。

> 💡 **记忆类比**：Kerberos 就像游乐园系统。KDC 是售票处，TGT 是手环/通票，TGT 证明你买过票，凭借手环可以去各个项目换取特定项目的票据。

#### AAA 协议对比
| 特性 | RADIUS | TACACS+ |
|---|---|---|
| **传输协议** | UDP（可靠性低） | TCP 端口 49（可靠性高） |
| **加密范围** | 仅加密密码 | 加密所有身份验证信息 |
| **AAA 处理** | 合并处理 | 将身份验证、授权和计费分离为独立进程 |
| **开发/标准** | 开放标准 | 思科开发，开放标准发布 |

### 14.3 零信任访问策略执行

*   **零信任**：假设不存在信任边界和网络边缘，每次操作请求都需作为持续身份验证过程进行验证，检查身份、权限、系统配置等策略后才允许访问。
*   **策略引擎**：使用信任算法，基于规则和外部系统做出授予、拒绝或撤销访问的策略决定。
*   **策略管理员**：根据策略引擎决定采取行动，建立或移除通信路径，创建特定于会话的验证令牌或凭据。
*   **策略决策点 (PDP)**：策略引擎和策略管理员共同组成的逻辑实体统称。
*   **策略执行点 (PEP)**：与策略管理员通信转发请求，接收允许或终止连接指令，控制不受信任系统向企业资源发起的连接。

> 💡 **记忆类比**：PEP 是门口的保安（执行动作），PDP 是后台的审批中心（包含大脑策略引擎和发通行证的管理员）。

### 14.4 了解访问控制攻击

#### 特权升级
*   **水平特权升级**：获取其他账户相似的特权（横向移动）。
*   **垂直特权升级**：获取显著更大的特权（如普通用户提升为管理员）。

#### 密码攻击类型
| 攻击类型 | 英文 | 攻击机制 |
|---|---|---|
| **字典攻击** | Dictionary Attack | 使用预定义数据库或常见密码列表尝试。 |
| **暴力攻击** | Brute-Force Attack | 系统尝试字母、数字和符号的所有可能组合。 |
| **喷射攻击** | Spraying Attack | 使用**同一个**猜测密码在**不同账户**列表循环，以绕过账户锁定。 |
| **凭证填充** | Credential Stuffing | 使用被盗数据库的凭证在其他网站自动化尝试。 |

> ⚠️ **考试陷阱**：喷射攻击的核心是"一个密码试多个账号"（防锁定），而常规暴力/字典攻击是"多个密码试一个账号"。

#### 哈希与密码破解攻击
*   **生日攻击**：专注于寻找哈希碰撞，基于生日悖论，不需所有样本即可高概率发现相同哈希值。
*   **彩虹表攻击**：使用预计算哈希值的大型数据库直接比对，显著减少破解时间。
*   **哈希传递攻击**：直接发送捕获的密码哈希值模拟登录，无需知道明文密码。
*   **传递哈希 / 传递密钥**：在禁用 NTLM 的网络中，使用哈希值请求 TGT 访问资源。
*   **传递票据**：收集 `lsass.exe` 进程中的票据并注入以冒充用户。

#### Kerberos 特定攻击
*   **白银票据**：使用服务账户 NTLM 哈希伪造 TGS 票据，获取该服务账户的所有特权。
*   **黄金票据**：获取 KRBTGT 账户哈希，可随意伪造 Kerberos 票据并为任何服务请求 TGS。

> ⚠️ **关键区分**：黄金票据 = KRBTGT 哈希 = 整个域的上帝模式；白银票据 = 服务账户哈希 = 仅该服务特权。

#### 凭证保护机制
*   **盐**：计算哈希前添加的随机位，增加彩虹表及离线字典/暴力攻击难度。
*   **胡椒**：存储在数据库之外的大型常量数字，进一步提高安全性。

> 💡 **记忆类比**：盐是每份密码独有的随机调料（存在数据库里），胡椒是所有密码共用的秘方（存在代码/配置里，不与哈希同库存）。

#### 其他攻击工具与方式
*   **Mimikatz**：从内存提取明文密码/PIN、Kerberos 票据、证书和私钥的工具。
*   **嗅探攻击**：使用嗅探器捕获并明文读取网络传输数据。
*   **欺骗攻击**：假装成其他事物或人（如伪造 IP 地址或发件人字段）。
