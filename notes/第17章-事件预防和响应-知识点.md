# 第17章 事件预防和响应

### 1.1 事件管理基础

#### 事件
* **定义**：指对组织资产的机密性、完整性或可用性产生负面影响的任何事件，包括直接攻击、自然事件以及意外事故。

#### 事件管理
* **主要目标**：尽量减少事件对组织造成的影响。

#### 计算机安全事件响应团队 (CSIRT / CIRT)
* **定义**：在重大安全事件发生时被激活的专门团队。
* **职责**：调查事件、遏制和评估损失、收集证据、报告事件并执行恢复程序。

#### 易失性数据保护
* **定义**：在遏制事件时保护临时文件和RAM中的数据证据。
* ⚠️ **考试陷阱/关键操作**：在遏制事件时**绝对不得关闭计算机电源**，否则将丢失RAM和临时文件中的易失性证据。

### 1.2 事件响应生命周期

#### 检测
* **定义**：通过入侵检测系统、防恶意软件、自动日志扫描或用户报告等多种方式发现潜在事件，并由IT人员调查确认其是否为安全事件的初始步骤。

#### 缓解
* **定义**：试图遏制事件以限制其影响或范围的步骤。
* **示例**：禁用网络适配器或断开网络连接以将问题隔离在单一网络内。

#### 报告
* **定义**：在组织内部（如向高层管理人员）以及向组织外部的机构和个人通报事件的过程。
* ⚠️ **合规要求**：通常受旨在保护隐私和数据的监管合规法律约束。

#### 恢复
* **定义**：将系统恢复或退回到完全运行状态的步骤。
* **示例**：对于重大事件，可能需要从头开始完全重建系统并从最近的备份中恢复所有数据。

#### 补救
* **定义**：相关人员检查事件，确定导致事件发生的原因，随后实施方法（如修补漏洞）以防止此类事件再次发生的阶段。

#### 根本原因分析
* **定义**：在补救阶段执行的一种检查分析方法，旨在确定允许事件发生的根本漏洞或系统缺陷。

#### 总结教训
* **定义**：检查事件及响应过程以寻找响应能力改进空间的阶段。
* **反馈机制**：该阶段的输出结果可反馈至事件管理的第一步以提升检测能力并帮助防止事件重演。

#### 个人身份信息数据泄露通知
* ⚠️ **法律要求**：如果数据泄露暴露了个人身份信息（PII），大多数法律要求组织必须报告该事件，并通知受事件影响的个人。

### 2.1 安全控制类型

#### 预防性控制
* **定义**：试图阻挠或阻止不必要的或未经授权的活动发生。

#### 检测性控制
* **定义**：试图在不必要的或未经授权的活动发生后，去发现或检测这些活动。

### 2.2 常见攻击类型

#### 僵尸网络
* **定义**：由多个被称为“机器人”或“僵尸”的受感染计算机组成的网络，由控制者通过命令和控制服务器指挥以执行如DDoS攻击或发送垃圾邮件等指令。

#### 拒绝服务攻击
* **定义**：来自单一系统并针对单一系统，旨在阻止目标系统处理或响应合法的流量或资源请求。

#### 分布式拒绝服务攻击
* **定义**：多个系统（如僵尸网络）在同一时间联合攻击单一目标系统。

#### 分布式反射型拒绝服务攻击
* **定义**：DoS的一种变体，不直接攻击受害者，而是操纵流量或网络服务，使攻击从其他来源反射回受害者。

#### SYN 洪水攻击
* **定义**：通过向目标系统发送大量SYN数据包且从不完成带有ACK的连接，破坏TCP标准的三次握手，从而耗尽系统的可用内存和处理能力。

#### TCP 重置攻击
* **定义**：通过在RST（重置）数据包中伪造源IP地址来恶意断开目标系统间活跃的TCP会话。

#### Smurf 攻击
* **定义**：使用受害者的IP地址作为源IP地址，向放大网络发送伪造的广播Ping（ICMP回显请求），导致网络上所有系统向受害者发送回显回复的拒绝服务洪水攻击。

#### Fraggle 攻击
* **定义**：类似于Smurf攻击，但不使用ICMP，而是使用伪造受害者IP地址的UDP数据包（通过UDP端口7或19）进行网络广播。

#### Ping 洪水攻击
* **定义**：通过向受害者系统发送大量的Ping请求（ICMP回显请求），使其无法有时间响应合法请求。

#### 死亡之 Ping (Ping of Death)
* **定义**：利用超大的Ping数据包导致某些操作系统崩溃或发生缓冲区溢出错误的传统攻击。

#### Teardrop 攻击
* **定义**：将IP数据包进行碎片化，使接收系统难以或无法重新组装它们，从而导致系统崩溃。

#### LAND 攻击
* **定义**：向受害者发送伪造的SYN数据包，且该数据包的源IP和目的IP地址均为受害者的IP地址。

#### 零日漏洞利用
* **定义**：利用不为他他人知，或者供应商尚未开发或发布补丁的系统漏洞进行的攻击。

#### 中间人攻击
* **定义**：恶意用户在正在进行的通信的两个端点之间建立位置，以拦截、捕获、转发或更改系统之间交换的数据。

#### 员工蓄意破坏
* **定义**：由对组织资产有足够了解且拥有操纵环境权限的员工（通常是不满的被解雇员工）对组织进行的破坏或中断的犯罪行为。

🧠 **DoS攻击记忆矩阵**：

| 攻击类型 | 核心机制 | 关键协议/特征 |
| :--- | :--- | :--- |
| **DoS** | 单对单耗尽资源 | 单一源，单一目标 |
| **DDoS** | 多对单耗尽资源 | 僵尸网络，多源攻击单一目标 |
| **DRDoS** | 借力打力（反射） | 伪造受害者源IP，利用第三方反射流量 |
| **SYN Flood** | 半开连接耗尽资源 | TCP三次握手不完成（缺ACK） |
| **TCP Reset** | 强制断开连接 | 伪造RST包 |
| **Smurf** | 广播放大攻击 | ICMP回显请求 |
| **Fraggle** | 广播放大攻击 | UDP端口7或19 |
| **Ping Flood** | 洪水淹没 | 大量ICMP回显请求 |
| **Ping of Death** | 缓冲区溢出 | 超大Ping包 |
| **Teardrop** | 碎片重组错误 | 偏移量错误的IP碎片 |
| **LAND** | 自循环攻击 | 源IP=目的IP的SYN包 |

### 2.3 入侵检测与预防系统 (IDS/IPS)

#### 入侵检测系统 (IDS)
* **定义**：通过自动检查日志和实时系统事件，来检测表明潜在事件或入侵的异常活动。

#### 基于知识的检测 (Knowledge-Based Detection / 基于特征的检测)
* **定义**：使用已知攻击模式的数据库来匹配实时流量。
* **优点**：误报率低。
* **缺点**：只能有效应对已知攻击。

#### 基于行为的检测 (Behavior-Based Detection)
* **定义**：通过创建正常活动的基线，并利用启发式评估技术与当前活动进行比较来检测恶意异常。
* **优点**：能检测新型攻击。
* **缺点**：通常会产生大量误报。

#### 检测结果判定矩阵
* **假阳性 / 误报**：系统检测到了事件，但实际上并未发生真正的事件或攻击。
* **假阴性 / 漏报**：确实发生了真实的事件或攻击，但系统未能将其检测出来。⚠️ **最危险的检测状态**。
* **真阳性**：发生了一个真实事件，并且系统成功地检测到了它。
* **真阴性**：没有发生任何事件，系统也没有检测到任何内容。

🧠 **检测判定类比**：想象火灾警报器。
* 真阳性 = 起火了，警报响了。
* 真阴性 = 没起火，警报没响。
* 假阳性（误报）= 没起火（只是烤面包冒烟），警报响了。
* 假阴性（漏报）= 起火了，警报没响（致命！）。

#### 被动响应
* **定义**：IDS在检测到事件时，仅记录该事件并发送通知给管理员，而不改变环境。

#### 主动响应
* **定义**：除了记录和发送通知外，还通过修改环境（例如修改防火墙ACL以阻止特定流量）来主动阻止活动的响应方式。

#### 基于主机的IDS (HIDS)
* **定义**：监控单台计算机（包括进程调用和日志）上的活动。
* **优点**：能详细检查本地事件并发现网络IDS无法检测到的本地异常。
* **缺点**：消耗宿主系统资源且管理成本较高。

#### 基于网络的IDS (NIDS)
* **定义**：利用放置在关键网络位置的远程传感器，通过观察网络流量模式来集中监控并评估整个网络的活动以检测攻击。

#### 入侵预防系统 (IPS)
* **定义**：一种串联部署在网络流量中的特殊主动IDS。
* **机制**：所有流量必须通过它，使其能够在攻击到达目标系统之前检测并选择性地拦截攻击。

#### TLS 解密器
* **定义**：通常是串联部署的专用硬件设备，用于检测TLS流量并将其解密，随后将解密后的流量发送给IDPS进行深度检查。

### 2.4 诱捕与访问控制技术

#### 蜜罐
* **定义**：作为陷阱或诱饵创建的单个计算机，通常故意配置有漏洞以吸引入侵者攻击，从而保护真实网络并允许管理员观察攻击者的活动。

#### 蜜网
* **定义**：将两个或多个联网的蜜罐设备结合在一起使用，以模拟和伪装成一个完整的合法网络环境。

#### 伪漏洞
* **定义**：系统中有意植入的虚假漏洞或表面漏洞，旨在诱惑攻击者尝试利用，从而触发警报机制让管理员察觉威胁。

#### 诱惑 vs. 诱捕
* **诱惑**：合法的安全策略。指在没有主动招揽的情况下，将带有已知漏洞的系统（如蜜罐）放置在网络上，由攻击者自行发现并自愿决定执行未授权操作。
* **诱捕**：非法的行为。指系统所有者主动招揽、欺骗或鼓励访问者执行非法或未经授权的操作，随后指控其非法入侵。
* ⚠️ **考试陷阱**：务必区分**诱惑（合法）**与**诱捕（非法）**。蜜罐的使用属于诱惑，只要你不主动引诱黑客去攻击它。

#### 警告横幅
* **定义**：通过向用户明确声明禁止未经授权的访问并且所有活动都将被监控和记录，从而在法律上约束用户的行为准则。

#### 白名单 / 允许列表
* **定义**：标识了被授权在系统上运行的应用程序列表，并默认阻止所有不在列表上的其他应用程序运行。

#### 黑名单 / 拒绝列表
* **定义**：明确标识了未经授权且被禁止在系统上运行的特定应用程序列表。

#### Web 应用防火墙 (WAF)
* **定义**：一种专门的应用程序防火墙，通过深度检查流向Web服务器的流量来拦截SQL注入攻击和跨站脚本（XSS）攻击等恶意流量。

#### 下一代防火墙 (NGFW)
* **定义**：充当统一威胁管理（UTM）设备，结合了传统防火墙的数据包过滤、状态检测功能以及深度包检查、恶意软件过滤和入侵检测与预防能力。

#### 沙箱
* **定义**：一种虚拟化技术，它为应用程序提供了一个受隔离的安全边界，防止该应用程序（或正在测试的未知代码）与操作系统或其他应用程序进行交互。

### 3.1 日志与审计基础

#### 日志记录
* **定义**：将有关事件的信息记录到日志文件或数据库中的过程。它捕获事件、更改、消息以及描述系统上活动的其他数据。

#### 审计
* **定义**：为确保合规性而检查或视察环境，它比单纯的日志记录包含更多内容。

#### 审计追踪
* **定义**：当有关事件和发生情况的信息存储在一个或多个数据库或日志文件中时创建的记录。
* **作用**：提供系统活动的记录，并且能够重建安全事件发生前和发生期间的活动。

### 3.2 日志类型

| 日志类型 | 记录内容 |
| :--- | :--- |
| **安全日志** | 对文件、文件夹和打印机等资源的访问情况 |
| **系统日志** | 系统启动/停止、服务启动/停止、服务属性被修改的时间 |
| **应用程序日志** | 特定应用程序的信息（由开发人员决定记录内容） |
| **防火墙日志** | 到达防火墙的流量（允许和阻止），源/目标IP及端口（**不含数据包实际内容**） |
| **代理日志** | 特定用户访问的网站、停留时间、尝试访问违禁网站的时间 |
| **变更日志** | 系统变更请求、审批以及实际发生的变更（属于变更管理过程） |

### 3.3 监测与分析

#### 监测
* **定义**：审查信息日志以寻找特定内容的过程。
* **目的**：确保主体（如用户和员工）对其行动和活动负责。

#### 调整
* **定义**：调节安全控制措施以更好地匹配组织及其运行环境需求的过程。

#### 日志分析
* **定义**：一种详细且系统的监测形式，通过分析记录的信息来发现趋势和模式，以及异常、未经授权、非法和违反策略的活动。

### 安全信息与事件管理

#### SIEM (Security Information and Event Management)
*   **定义**：一种集中式应用程序，用于提供集中日志记录并对整个组织系统上发生的事件进行实时分析。
*   **核心功能**：关联和聚合，将从不同设备收集的数据转换为有用的信息。

#### Syslog协议
*   **定义**：由 RFC 5424 描述，用于向集中的 Syslog 服务器发送事件通知消息的协议。
*   **功能边界**：定义了如何**格式化和发送**消息，但**未定义如何处理**它们。
*   ⚠️ **考试陷阱**：注意 Syslog 只管“发”不管“处理”，不要混淆其与 SIEM 的处理能力。

#### 日志管理
*   **定义**：用于收集、处理和保护日志条目的所有方法。

#### 循环日志记录
*   **定义**：允许管理员设置最大日志大小，当日志达到该大小时，系统开始覆盖日志中最旧的事件。

### 数据提取与监控技术

#### 取样 / 数据提取
*   **定义**：从大量数据收集中提取特定元素，以构建整体的有意义的表示或摘要的过程。

#### 统计取样
*   **定义**：使用精确的数学函数从大量数据中提取有意义的信息，并**可以确定误差幅度**。

#### 剪切级别
*   **定义**：一种**非统计**取样形式，只选择超过预定义阈值的事件，系统会忽略未达到此阈值的常规事件。
*   ⚠️ **易错点**：剪切级别属于**非统计**取样，且无法确定误差幅度（与统计取样区分）。

#### 击键监控
*   **定义**：记录用户在物理键盘上执行的击键动作的行为，通常通过硬件设备或键盘记录器软件完成。

#### 流量分析和趋势分析
*   **定义**：检查数据包**流向**而非数据包**实际内容**的监控形式，有时也被称为网络流监控。
*   💡 **记忆类比**：就像监控信封上的寄信/收信地址（流向），而不拆开信封看信件内容。

#### 出口监测
*   **定义**：监控离开网络前往互联网的流量，以检测未经授权的数据传出（数据外泄）。

#### 数字水印
*   **定义**：隐藏在数字文件中的秘密标记。
*   **与 DLP 协同**：DLP 系统可检测未加密文件中的水印，识别敏感数据时阻止传输并报警。
*   ⚠️ **边缘情况**：数字水印通常在**未加密**文件中才能被 DLP 系统有效检测。

### 自动事件响应 (17.4)

#### SOAR (Security Orchestration, Automation, and Response)
*   **定义**：允许组织自动响应某些安全事件的技术。
*   **运作方式**：通常使用**剧本**和**运行手册**来定义安全事件及对应的响应措施。

#### 剧本 vs. 运行手册
| 概念 | 定义与作用 | 核心特点 |
| :--- | :--- | :--- |
| **剧本** | 定义如何验证安全事件及响应操作详细信息的文档/检查清单。 | 记录运行手册应执行的操作；作为自动化失败时管理员**手动完成步骤的备份参考**。 |
| **运行手册** | 将剧本中定义的数据和操作落实到自动化工具中实施的组件。 | 执行条件步骤验证威胁，确认攻击后**自动执行**指定操作缓解威胁。 |
*   💡 **记忆口诀**：**剧本是“人”看的指南，运行手册是“机器”跑的代码。**

#### 机器学习 (ML) vs. 人工智能 (AI)
| 概念 | 起始状态 | 学习机制 |
| :--- | :--- | :--- |
| **机器学习 (ML)** | 从一组**规则或基线**开始 | 根据接收到的反馈（如有效警报和误报）持续修改和学习 |
| **人工智能 (AI)** | 从**零知识**（无规则或基线）开始 | 通过自我监控和观察逐渐学习规则，创建自己的算法与基线 |
*   ⚠️ **常见误区**：AI 不是更高级的 ML。核心区别在于**起点**：ML 需要预设基线，AI 从零开始自我构建规则。

### 威胁情报与威胁狩猎

#### 威胁情报
*   **定义**：收集有关潜在威胁的数据，包括使用各种来源及时获取当前威胁信息，以用于搜寻威胁。

#### 威胁订阅源 vs. 威胁情报订阅源
| 概念 | 内容性质 | 包含内容示例 |
| :--- | :--- | :--- |
| **威胁订阅源** | **原始数据**的稳定流 | 可疑域名、已知恶意软件哈希值、共享代码、恶意 IP 地址 |
| **威胁情报订阅源** | **可供采取行动的** 情报流 | 从原始数据中提取、分析得出的可操作情报 |
*   💡 **记忆类比**：威胁订阅源是“未经加工的原油”，威胁情报订阅源是“提炼后可直接加注的汽油”。

#### 杀伤链 vs. MITRE ATT&CK 矩阵
| 模型 | 性质 | 特点 |
| :--- | :--- | :--- |
| **杀伤链** | **有序的**攻击阶段集合 | 7个阶段：侦察、武器化、投递、利用、安装、命令与控制、达成目标的行动。防御方只要阻断任一阶段，整个攻击链即被破坏。 |
| **MITRE ATT&CK** | **动态化**的 TTPs 知识库 | 战术**不是有序的**，以矩阵形式列出，每年至少更新两次。 |
*   ⚠️ **考试陷阱**：杀伤链是**线性/有序**的，ATT&CK 是**非线/动态**的，切勿混淆两者的有序性。

#### 威胁狩猎
*   **定义**：在网络中**主动**搜索网络威胁的过程，超越被动等待工具检测的方式。
*   **核心前提**：假设**攻击者已经存在于网络中**，由安全专业人员主动搜寻威胁指标。

#### 误报 vs. 漏报
| 概念 | 实际情况 | 系统表现 |
| :--- | :--- | :--- |
| **误报** | 安全问题**不存在** | 系统发出警报指示存在问题 |
| **漏报** | 安全问题/攻击**存在** | 未被检测系统发现 |
*   💡 **记忆口诀**：误报是“狼来了”（没狼喊狼），漏报是“掩耳盗铃”（有狼没看见）。
