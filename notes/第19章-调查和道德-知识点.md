# 第19章-调查和道德-知识点

### 19.1 调查

#### 调查类型
| 调查类型 | 核心定义 | 证据标准/目的 |
| :--- | :--- | :--- |
| **行政调查** (Administrative) | 内部调查，检查运营问题或违反组织策略的行为 | 技术排障或支持HR纪律处分 |
| **运营调查** (Operational) | 检查与组织计算基础设施相关的问题 | 解决运营问题，**不是为了产生证据** |
| **刑事调查** (Criminal) | 执法人员调查涉嫌违反刑法的行为 | **排除合理怀疑** (Beyond a reasonable doubt) |
| **民事调查** (Civil) | 准备证据以在民事法庭解决双方争议 | **证据优势** (Preponderance of the evidence) |
| **监管调查** (Regulatory) | 政府机构调查个人或公司违反行政法的行为 | 与审理该案的场所相称 |

⚠️ **考试陷阱**：注意区分运营调查与行政调查——运营调查的核心目标是**解决运营问题**且**不产生证据**；而行政调查常用于支持内部纪律处分。另外，刑事与民事调查的证据标准极易混淆，刑事要求极高的"排除合理怀疑"，民事仅需"证据优势"（即更有可能发生，>50%）。

#### 电子发现
*   **电子发现**：在法律程序中为披露目的而促进电子信息的处理过程。
*   **电子发现参考模型 (EDRM)**：包含9个方面的标准过程：
    1. 信息治理
    2. 识别
    3. 保存
    4. 收集
    5. 处理
    6. 审查
    7. 分析
    8. 生产
    9. 展示

🧠 **记忆辅助**：EDRM九步首字母缩写 **I** **G** **I** **P** **C** **P** **R** **A** **P** (识别、治理、识别、保存、收集、处理、审查、分析、生产/展示) —— 按实际流程顺序记忆更佳：治理先行，然后识别、保存、收集、处理、审查、分析、生产、展示。

#### 证据类型与规则
*   **可接受的证据**：必须满足三个基本要求：
    1. **相关** (Relevant)：与确定事实相关
    2. **实质** (Material)：对案件具有实质意义
    3. **合法** (Competent)：合法获取的

⚠️ **常见误区**：Competent 在日常英语中常指"胜任"，但在法律证据语境中指"合法获取的"，切勿混淆。

| 证据类型 | 核心定义 | 示例 |
| :--- | :--- | :--- |
| **实物证据** (Real) | 可以实际带上法庭的物理物品 | 被扣押的计算机、带指纹的键盘、硬盘 |
| **书面证据** (Documentary) | 证明当前事实的、经过验证的书面材料 | 由证人确认为日常业务记录的计算机日志 |
| **证言证据** (Testimonial) | 证人宣誓后口头或书面提供的证词 | 证人必须具有其证词所依据的个人知识 |
| **演示证据** (Demonstrative) | 支持证言证据，帮助解释概念或澄清问题 | 解释网络数据包的图表、DDoS攻击过程演示 |

#### 重要证据规则
*   **最佳证据规则**：法庭程序中必须出示**原始文件**，不接受副本或描述（次要证据），除非适用特定例外。
*   **口头证据规则**：当协议以书面确立后，书面文件被假定包含所有条款，**口头协议不得修改**书面协议。
*   **传闻规则**：禁止证人就其他人在庭外告诉他们的事情作证（无法证实可靠性）。
*   **业务记录例外**：传闻规则的重要例外。允许业务记录（如系统日志）作为证据，前提是：由具有直接知识的人/事物在事件发生时制作，且保留记录是组织的常规做法。

⚠️ **考试陷阱**：计算机生成的日志通常属于传闻，但常通过**业务记录例外**被采纳为证据。需牢记其两大前提：发生时制作 + 常规做法保留。

#### 取证与调查程序
*   **证据链**：全面记录证据从收集到出庭期间所有接触人员、时间及访问原因的日志，确保原件未被篡改。
*   **介质分析**：从磁性/光学等物理存储设备中识别和提取信息。
*   **写保护器**：硬件适配器，物理阻断向存储设备写入数据的线缆部分，防止意外篡改。
*   **取证镜像**：存储数据的**按位副本**。必须计算并验证加密哈希值以确保与原始介质完全相同，所有分析均在镜像上进行。
*   **网络分流器**：执行与交换机SPAN端口相同功能的硬件，镜像端口数据生成数据包转储，不改变实际网络流量。
*   **洛卡德交换原则**：法医/数字取证核心原则——**"每一次接触都会留下痕迹"**。

⚠️ **边缘情况**：取证镜像的哈希验证必须在创建镜像后立即进行，且后续分析绝对不能在原始介质上进行。

#### 约谈与审讯
*   **约谈**：仅为协助调查收集信息的谈话。
*   **审讯**：怀疑某人涉嫌犯罪且打算将信息用于法庭的正式质询。

### 19.2 计算机犯罪的主要类别

*   **计算机犯罪**：涉及计算机的犯罪或违法行为，可针对计算机本身，或利用计算机实施犯罪。

| 攻击类别 | 核心动机/目标 |
| :--- | :--- |
| **军事和情报攻击** | 获取执法、军事或技术研究的机密/受限信息 |
| **高级持续性威胁** | 资金充裕、高技能攻击者代表民族国家/犯罪组织等发起的复杂、高度有效攻击 |
| **商业攻击** | 破坏企业信息的CIA（机密性、完整性、可用性）；收集竞争对手机密知识产权（企业/工业间谍） |
| **财务攻击** | 非法获取金钱或服务；雇佣网络犯罪（雇佣兵活动） |
| **恐怖攻击** | 破坏正常生活并灌输恐惧（⚠️区别于提取机密的军事/情报攻击） |
| **恶意攻击** | 对组织/个人造成损害，动机通常是怨恨（现任/前任员工或心怀恶意者） |
| **内部威胁** | 对计算资产构成**最大风险**；包括恶意内部人员及特权员工失误危及安全 |
| **兴奋攻击** | 仅为好玩，动机是侵入系统带来的"快感" |
| **脚本小子** | 缺乏自行设计攻击能力，仅下载运行他人程序/脚本发起攻击 |
| **黑客行动主义者** | 结合政治动机与黑客刺激，破坏哲学理念上存在分歧的组织 |

⚠️ **考试陷阱**：内部威胁常被误认为仅指恶意的内部人员，实际上**拥有特权访问权限员工的意外失误**也属于内部威胁，且通常构成最大风险。恐怖攻击与军事攻击的目标不同（恐惧 vs 机密）。

### 19.3 道德规范

*   **道德规范**：指导日常活动的道德准则和个人行为规范，代表专业行为的**最低标准**，而不是法律。

#### ISC2 道德规范
由一个序言和四项条款组成，严格遵守是获得认证的先决条件：
1.  **保护社会**、共同利益、必要的公众信任和信心以及基础设施。
2.  **行为端正**、诚实、公正、负责且守法。
3.  为委托人提供**勤勉尽责和胜任**的服务。
4.  **推进和保护**本职业。

🧠 **记忆辅助**：ISC2四项条款首字缩写 **P**rotect, **A**ct, **P**rovide, **A**dvance (PAPA) —— 保护社会、行为端正、提供服务、推进职业。

#### ISC2 道德规范投诉
*   只有认为自己因涉嫌行为**受到伤害**的人才能提交正式投诉。
*   必须以**书面宣誓书**的形式提交。
*   必须指明被违反的**具体条款**。

#### RFC 1087《道德规范与互联网》
*   由互联网架构委员会 (IAB) 于1989年发布。
*   界定了不道德和不可接受的互联网行为：
    1. 寻求未经授权的访问
    2. 破坏预期用途
    3. 浪费资源
    4. 破坏基于计算机的信息完整性
    5. 损害用户隐私
