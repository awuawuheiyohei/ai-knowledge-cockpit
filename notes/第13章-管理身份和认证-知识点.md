# 第13章 管理身份和认证

### 13.1 控制对资产的访问

#### 物理安全控制
*   **定义**：可以触摸到的控制措施，通过控制访问和环境来保护系统、设备和设施。
*   **范围**：周界安全控制（栅栏、大门、警卫、旋转门）、环境控制（暖通空调系统、消防系统）。

#### 逻辑访问控制
*   **定义**：用于保护对信息、系统、设备和应用程序访问的技术控制措施。
*   **范围**：包括身份验证、授权和权限，防止未经授权访问数据及配置设置。

#### CIA三要素与访问控制
| 概念 | 定义 | 丧失的体现 |
| :--- | :--- | :--- |
| **机密性** | 确保只有经过授权的主体才能访问客体。 | 未经授权的实体能够访问系统或数据。 |
| **完整性** | 确保数据或系统配置不被未经授权修改，或发生未授权更改时能被检测到。 | 对客体发生了未经授权或不必要的更改。 |
| **可用性** | 对客体的授权请求在合理时间内授予主体，系统和数据在需要时可供使用。 | 系统无法运行或数据无法访问。 |

### 13.2 AAA模型与身份验证因素

#### AAA模型核心概念
*   **AAA模型**：身份和访问管理系统的核心功能，包含**身份认证**、**授权**和**问责**。
*   **身份识别**：主体声称或表明身份的过程，系统中所有主体必须具有**唯一**的身份。
*   **身份认证**：通过将一个或多个因素与有效身份数据库进行比较来验证主体的身份。
*   **授权**：基于已证明的身份向主体授予对客体的访问权限，确保请求符合分配的权限。
*   **问责**：通过审计、日志记录和监控提供，跟踪主体访问客体并创建审计跟踪。

#### 主体与客体
*   **主体**：访问被动客体以接收信息的**活动实体**（如用户、程序、进程、服务、计算机）。
*   **客体**：向活动主体提供信息的**被动实体**（如文件、数据库、计算机、程序、打印机、存储介质）。

#### 身份验证因素
| 类型 | 名称 | 包含内容/示例 |
| :--- | :--- | :--- |
| **Type 1** | 你知道什么 | 用户记住的秘密（密码、PIN、密码短语） |
| **Type 2** | 你拥有什么 | 物理对象（智能卡、硬件令牌、智能手机） |
| **Type 3** | 你是什么 | 人的物理特征/生物识别（指纹、面部扫描、语音模式） |
| - | 你在哪里 | 特定计算机、IP地址标识的地理位置、呼叫者ID标识的电话号码 |
| - | 上下文感知 | 识别用户位置、时间、移动设备等多个属性（常用于MDM） |

> 💡 **记忆 aid**：验证三要素口诀 —— **KHA** (Know, Have, Are) / **知、拥、是**。

#### 基于知识的身份验证 (KBA) 与认知密码
*   **KBA**：常用于对新用户进行身份证明，系统在创建问题前查询独立的权威数据源。
*   **认知密码**：用于用户尝试更改密码时的问题，答案在账户初始注册期间收集。

#### 密码管理策略
*   **密码最大使用期限**：要求用户定期更改密码（如每45天），又称密码过期。
*   **密码历史**：记住一定数量的先前密码，防止用户重复使用。

#### 令牌与智能卡
*   **智能卡**：信用卡大小，含嵌入式集成电路芯片（微处理器和证书），用于识别或验证。
*   **身份验证器/令牌**：随身携带的验证机密生成设备/程序，通常显示一次性密码（OTP）。
*   **TOTP (基于时间的一次性密码)**：同步验证器，由与服务器同步的设备/应用周期性生成，**要求准确且同步的时钟**。
*   **HOTP (基于哈希的一次性密码)**：异步验证器，不使用时钟，基于算法和**递增计数器**生成。

#### 生物识别错误率与评估
*   **错误拒绝率 (FRR / Type I error)**：未能对有效用户进行验证的比率。
*   **错误接受率 (FAR / Type II error)**：错误地对某人进行验证的比率。
*   **交叉错误率 (CER / ERR)**：FRR和FAR百分比相等的点。**CER越低的设备越准确**。
*   **吞吐率**：系统扫描主体并批准或拒绝访问所需的时间量。

> ⚠️ **考试陷阱**：极易混淆 Type I 和 Type II 错误。记住：**I = 拒绝**，**II = 接受**。比较生物识别设备准确性时，**CER越低越好**。

#### 多因素与高级认证
*   **多因素身份认证 (MFA)**：使用两个或多个**不同类型（因素）**的验证。
    > ⚠️ **常见误区**：输入密码+PIN码**不是**MFA，因为它们同属Type 1（你知道什么）。密码+智能卡才是MFA。
*   **无口令身份认证**：无需输入密码，FIDO联盟创建了围绕使用硬件通行密钥的框架和协议标准。
*   **设备指纹识别**：注册时捕获设备特征（OS、浏览器、分辨率等）用于后续验证。
*   **服务账户**：为服务/应用程序创建，通常高权限、强密码、配置为非交互式（防止用户传统登录）。
*   **双向身份认证**：客户端和服务器都提供验证，防止客户端向流氓服务器泄露信息。

### 13.3 实施身份管理

#### 访问控制模式
*   **集中式访问控制**：系统内由单一实体执行所有授权验证。
*   **分散式访问控制**：多个实体分布在系统各位置执行授权验证（又称分布式访问控制）。

#### 单点登录 (SSO) 与目录服务
*   **单点登录 (SSO)**：集中式技术，主体一次验证后即可访问多个资源，无需再次验证。
*   **目录服务**：集中式数据库，包含主体和对象信息（含验证数据）。
*   **安全域**：共享通用安全策略的主体和对象集合，可独立运行。
*   **信任**：域之间建立的连接，创建安全桥梁，允许一个域用户访问另一个域的资源。

#### 联合身份管理 (FIM)
*   **身份管理**：对用户身份及其凭证的管理。
*   **联合身份管理 (FIM)**：SSO的一种形式，将用户在一个系统的身份与多个IdM系统链接，允许多组织共享身份信息。
*   **基于云的联合**：使用第三方服务，用登录ID检索匹配联合身份并授权访问。
*   **本地联合**：组织内部/本地托管，允许不同网络（如合并的公司）共享验证数据并互访资源。
*   **混合联合**：基于云和本地解决方案的组合。

#### 其他身份管理机制
*   **准时制配置 (JIT)**：自动在两实体间创建关系，新用户首次访问时自动创建账户，无需管理员干预。
*   **凭证管理系统**：存储用户名密码，提供保存凭据、跳过登录表单、自动登录功能。
*   **身份即服务**：第三方提供的IAM服务，为云资源和SaaS应用有效提供SSO。
*   **密码保管库**：第三方凭证管理系统，将凭证存于加密数据库，通过主密码解锁。
*   **脚本访问**：登录会话开始时自动传输凭据的自动化过程，用于**模拟**SSO（非真正SSO技术）。
*   **会话管理**：防止未授权访问，如密码保护屏保、空闲超时后终止会话并强制重新验证。

### 13.4 管理身份和访问配置生命周期

#### 生命周期概述
*   **身份和访问配置生命周期**：账户的创建、管理、审查/审计和删除过程。

#### 生命周期阶段
*   **配置和入职**：创建用户账户，并配置新员工工作所需的所有特权。
*   **注册/登记**：新用户账户的初始创建过程，必须提供的唯一信息是用户名或唯一标识符。
*   **取消配置/账户撤销**：最简单方法是删除账户，移除员工在职期间的所有访问权限。
*   **账户维护**：生命周期中的持续维护，主要涉及更改权利和特权。
*   **账户访问审查**：管理员定期审查账户（用户、特权、系统、服务账户），确保无过度特权并符合安全策略。

#### 特权管理风险
*   **过度特权**：用户拥有的特权超过其工作任务所需，违反最小特权原则。
*   **特权蔓延**：随着工作角色和任务变化，用户账户不断累积额外特权的现象，违反最小特权原则。

> ⚠️ **考试陷阱**：过度特权是状态，特权蔓延是过程。两者都违反**最小特权原则**。账户审查是解决和预防特权蔓延的关键措施。
