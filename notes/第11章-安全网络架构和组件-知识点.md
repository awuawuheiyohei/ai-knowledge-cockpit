### 11.1 OSI模型

#### 核心概念
*   **网络协议**: 定义数据如何通过网络媒介（双绞线、光纤、无线）传输的规则和限制。
*   **OSI参考模型**: 由ISO在20世纪70年代末开发，将网络任务分为七个层级的通用通信结构/标准。
*   **TCP/IP协议**: 目前使用最广泛的协议，基于DARPA模型开发于20世纪70年代初。

#### 数据传输机制
*   **封装**: 数据从上层向下层传递前，添加头部（可能还有尾部）的过程。
*   **解封装**: 数据从物理层向应用层移动时，移除头部/尾部的相反过程（IETF使用的术语）。
*   **对等层通信**: 每一层移除的信息（指令、校验和等）只能由最初添加该信息的对等层理解。

#### 协议数据单元 (PDU)
| OSI层级 | PDU名称 | 说明 |
| :--- | :--- | :--- |
| 第7-5层 (应用/表示/会话) | 数据 | 网络数据容器的统称 |
| 第4层 (传输层) | 段 / 数据报 | TCP称为段，UDP称为数据报 |
| 第3层 (网络层) | 包 | 包含源/目标IP地址 |
| 第2层 (数据链路层) | 帧 | 包含源/目标MAC地址 |
| 第1层 (物理层) | 比特 | 转换后在物理介质上传输 |

> 💡 **记忆 aid (PDU顺序)**: **D**ata **S**egment **P**acket **F**rame **B**its (首字母记忆：**D**o **S**ome **P**ractical **F**ield **B**work / 数据段包帧比特)

#### OSI七层详解
*   **第7层 - 应用层**: 将用户应用程序、网络服务或操作系统与协议栈连接。
*   **第6层 - 表示层**: 数据格式转换（确保通用理解）、数据加密和压缩。
*   **第5层 - 会话层**: 建立、维护、终止通信会话，管理对话控制。
*   **第4层 - 传输层**: 管理连接完整性、控制会话、提供端到端传输服务。
*   **第3层 - 网络层**: 逻辑寻址和执行路由。
*   **第2层 - 数据链路层**: 格式化数据包，添加源/目标硬件地址（MAC地址）。
*   **第1层 - 物理层**: 将帧转换为比特进行物理传输（接收时反向操作）。

#### 通信模式
*   **单工**: 单向通信，设备只能发送或接收。
*   **半双工**: 双向通信，但同一时间只能单向传输。
*   **全双工**: 双向通信，可同时双向发送和接收。

#### 路由协议与自治系统
*   **距离矢量路由协议**: 内部路由协议，以**跳数**作为度量标准。
*   **链路状态路由协议**: 内部路由协议，根据路由器特征（速度、延迟、错误率、成本）决策。
*   **路径矢量路由协议**: 外部路由协议，根据到达目标的整个剩余路径决策。
*   **自治系统**: 在单一组织控制下，呈现统一路由策略的一组IP网络和路由器。

#### 网络硬件设备
*   **路由器**: 第3层设备，根据速度/跳数/偏好确定最佳逻辑路径。
*   **交换机与网桥**: 第2层设备，基于MAC地址路由流量。
*   **第1层硬件设备**: 网卡(NIC)、集线器、中继器、集中器、放大器（执行基于硬件的信号操作）。

### 11.2 TCP/IP模型

#### 核心概念
*   **TCP/IP模型**: 也称DARPA或DOD模型，仅由四个层级组成。

#### TCP/IP与OSI层级映射
| TCP/IP模型层级 | 对应的OSI模型层级 |
| :--- | :--- |
| 应用层 | 应用层、表示层、会话层 (5-7) |
| 传输层 | 传输层 (4) |
| 互联网层 | 网络层 (3) |
| 链路层 | 数据链路层、物理层 (1-2) |

#### TCP/IP脆弱性与攻击
*   **实施漏洞**: 缓冲区溢出、SYN泛洪、DoS攻击、分片攻击、超大数据包攻击、欺骗攻击、中间人、劫持攻击、编码错误攻击。
*   **被动攻击**: 极易受到监控或嗅探。

### 11.3 网络流量分析

*   **协议分析器**: 检查网络流量内容的框架/数据包捕获工具，能**解码和解释**详细内容。
*   **嗅探器**: 单纯的数据包/帧捕获工具，侧重收集流量，**解码解释能力弱**于协议分析器。
*   **混杂模式**: NIC忽略目的MAC地址，强制收集所有到达该接口的数据帧。
*   **捕获过滤器**: 预设规则，控制哪些被监听的数据帧被保存到文件/缓冲区，哪些被丢弃。

### 11.5 传输层协议

#### 端口与套接字
*   **端口**: 16位二进制数 (0-65,535)，用于传输层建立连接，支持IP复用。
*   **套接字**: IP地址 + 端口号的组合。

#### 端口分类
| 类别 | 范围 | 用途 |
| :--- | :--- | :--- |
| 知名端口/服务端口 | 0 - 1,023 | 保留供服务器使用 |
| 注册软件端口 | 1,024 - 49,151 | 由IANA注册给特定网络软件 |
| 随机/动态/临时端口 | 49,152 - 65,535 | 客户端随机临时用作源端口 |

> 💡 **记忆 aid (端口范围)**: **W**ell-known (1K以内), **R**egistered (1K-49K), **D**ynamic (49K-65K) 

#### TCP vs UDP
| 特性 | TCP | UDP |
| :--- | :--- | :--- |
| 连接性 | 面向连接 | 无连接 |
| 通信模式 | 全双工 | 单工 |
| 可靠性 | 可靠，保证数据交付 | 尽力而为，不可靠 |
| 错误/流量控制 | 支持排序和流量控制 | 无标准错误检测/纠正，无流量控制 |
| 开销与速度 | 开销大，速度较慢 | 开销极低，速度快 |
| 应用场景 | 要求保证交付 | 实时音视频/流媒体 |

#### TCP连接管理
*   **三次握手**: 建立会话。客户端发 `SYN` -> 服务器回 `SYN/ACK` -> 客户端发 `ACK`。
*   **FIN标志包**: 优雅地启动会话关闭（常见断开方法）。
*   **RST标志包**: 立即且突然地终止会话。

> ⚠️ **考试陷阱**: 题目常考UDP的"不可靠"特性与其"低开销/高速度"的权衡。不要误以为UDP完全无用，其实时性是关键优势。

### 11.6 域名系统

*   **域名**: 分配给IP地址的"临时"且便于人类记忆的命名约定。
*   **IP地址**: 分配在MAC地址之上/附加其上的"临时"逻辑地址。
*   **MAC地址**: (来自前文11.1) 6字节(48位)二进制硬件地址，十六进制表示，前3字节代表OUI。

> 💡 **类比记忆**: MAC地址像人的**身份证号**(永久/物理)，IP地址像人的**邮寄地址**(可变/逻辑)，域名像人的**名字**(方便记忆)。三者均具有"临时性"或"可变性"的相对特征，但MAC偏硬件，IP/域名偏逻辑。

### 域名系统 (DNS)

#### 核心概念
*   **硬件地址**: 被设计为“永久”物理地址的地址（如MAC地址）。
*   **域名系统 (DNS)**: 将便于人类记忆的完全限定域名 (FQDN) 解析并链接到其对应的等效 IP 地址的系统。
*   **地址解析协议 (ARP)**: 将 IP 地址解析为其对应的等效 MAC 地址的协议。
*   **完全限定域名 (FQDN)**: 由顶级域名 (TLD)、注册域名和子域名或主机名三个主要部分组成。总长度不超过 253 个字符，且任何单一区段不超过 63 个字符。
*   **区域文件**: 托管在权威名称服务器上的，关于特定域的资源记录或详细信息的集合。
*   **Hosts 文件**: 包含域名及其关联 IP 地址的硬编码引用的静态本地文件。

#### DNS 记录类型
| 记录类型 | 全称 | 功能定义 |
| :--- | :--- | :--- |
| **A** | Address Record | 将 FQDN 链接到 **IPv4** 地址 |
| **AAAA** | IPv6 Address Record | 将 FQDN 链接到 **IPv6** 地址 |
| **PTR** | Pointer Record | 将 IPv4/IPv6 地址链接到 FQDN（实现 **DNS 反向查找**） |
| **CNAME** | Canonical Name Record | 规范名称/别名记录，将一个 FQDN 链接到另一个 FQDN |
| **MX** | Mail Exchange Record | 标识域的 **SMTP** 电子邮件服务器 |
| **NS** | Name Server Record | 标识域的 DNS 服务器 |
| **SOA** | Start of Authority Record | 标识主要权威 DNS 服务器、负责的电子邮件地址、区域文件的序列号以及刷新、重试、过期和默认 TTL 的时间间隔 |

> 💡 **记忆 aid**: A/AAAA 记录是正向（名到IP），PTR 是反向（IP到名）。A 是 32位IPv4（4个A），AAAA 是 128位IPv6（6个A）。

#### DNS 安全扩展与加密
*   **域名系统安全扩展 (DNSSEC)**: 对现有 DNS 基础架构的安全改进，提供相互证书身份验证和加密会话，以减少区域文件中毒和 DNS 缓存中毒。
*   **基于 HTTPS 的 DNS (DoH)**: 为客户端创建与 DNS 服务器的 TLS 保护的 HTTP 加密会话，类似 VPN 形式保护 DNS 查询和响应。
*   **隐蔽型 DoH (ODoH)**: 在客户端和 DNS 解析器之间添加 DNS 代理以隔离请求客户端身份，提供匿名性和隐私性。

#### DNS 攻击与防御
*   **DNS 中毒**: 伪造客户端用于访问目标系统的 DNS 信息，通常返回虚假结果。
*   **流氓 DNS 服务器**: 侦听网络流量中与目标站点相关的 DNS 查询，并发送包含虚假 IP 信息的 DNS 响应。
*   **DNS 缓存中毒**: 将虚假/不正确的信息放入 DNS 服务器或客户端的本地 DNS 缓存中。
*   **DNS 劫持/域欺骗**: 通过修改本地 hosts 文件或毒化、欺骗 DNS 解析，将有效网站的 URL/IP 恶意重定向到虚假网站。
*   **DNS 查询欺骗**: 威胁行为者窃听客户端发出的查询，并发送回包含从原始查询中克隆的正确查询 ID (QID) 的虚假回复。
*   **域名劫持**: 在未经有效所有者授权的情况下恶意更改域名注册。
*   **同形异义字攻击**: 利用字符集（如拉丁字母与西里尔字母）的相似性来注册在肉眼看来合法的虚假国际化域名 (IDN)。

#### DNS 防御架构
*   **分裂 DNS (Split-DNS)**: 部署供公众使用的外部 DNS 服务器，以及仅允许内部系统访问的内部 DNS 服务器的安全架构。
*   **DNS 黑洞**: 用作防御机制的虚假遥测系统，对来自恶意软件的 DNS 查询提供虚假响应，阻止其访问 C2 控制系统，或保护用户免受已知恶意网站侵害。

### 互联网协议 (IP)

#### 核心概念
*   **互联网协议 (IP)**: 在 OSI 模型的网络层运行，提供数据包的路由寻址。是无连接且不可靠的“尽最大努力”通信服务，通常必须与 TCP 结合使用以获得可靠、受控的通信会话。
*   **无类别域间路由 (CIDR)**: 更灵活的子网划分系统，使用掩码位计数符号（如 /16）取代传统的点分十进制子网掩码，能将多个不连续的地址集合并为一个更大的单一子网。

#### IPv4 与 IPv6 对比
| 特性 | IPv4 | IPv6 |
| :--- | :--- | :--- |
| **地址长度** | 32 位 | 128 位 |
| **IPSec 支持** | 需要附加 IPSec | 原生支持 IPSec |
| **新特性** | 无 | 作用域地址、自动配置、QoS 优先级值 |

#### IPv6 自动配置与 DHCP
*   **DHCPv6**: 
    *   *有状态模式*: 分配特定 IPv6 地址并管理网络配置参数。
    *   *无状态模式*: 仅提供网络配置参数，不分配特定 IPv6 地址。
*   **无状态地址自动配置 (SLAAC)**: 设备生成自己的 IPv6 地址。机制：将路由器通告 (RA) 消息中的网络前缀 + 网卡 MAC 地址派生（或随机生成）的接口标识符结合。

#### IPv4 地址分类
| 类别 | 首部二进制位 | 第一个八位字节范围 | 默认子网掩码 | 最大主机数 |
| :--- | :--- | :--- | :--- | :--- |
| **A 类** | 0 | 1 - 126 | 255.0.0.0 (/8) | 16,777,214 |
| **B 类** | 10 | 128 - 191 | 255.255.0.0 (/16) | 65,534 |
| **C 类** | 110 | 192 - 223 | 255.255.255.0 (/24) | 254 |
| **D 类** | 1110 | 224 - 239 | (用于多播) | N/A |
| **E 类** | 1111 | 240 - 255 | (保留实验/未来使用) | N/A |

> ⚠️ **考试陷阱**: 127.x.x.x 属于 A 类范围，但被保留为环回地址，不可分配给主机。A 类有效主机网络范围是 1-126。

#### 特殊 IP 地址与协议
*   **环回地址**: IPv4 为 127 子网（通常 127.0.0.1）；IPv6 为 `::1/128` 或 `::1%128`。
*   **黑洞网络**: 0 网络（A 类网络为 0），路由到此网络的流量将被直接丢弃。
*   **互联网控制消息协议 (ICMP)**: 确定网络或特定链路的健康状况（ping, traceroute）。常被黑客利用发起死亡之 ping、Smurf 等带宽耗尽型 DoS 攻击。
*   **互联网组管理协议 (IGMP)**: 允许系统支持多播，管理主机动态的多播组成员身份。

#### IPv4 与 IPv6 共存机制
*   **双栈**: 系统同时运行 IPv4 和 IPv6。
*   **隧道传输**: 使用封装隧道访问另一种协议的系统。
*   **NAT-PT**: 在 IPv4 和 IPv6 网段之间进行网络地址转换和协议转换。

> ⚠️ **IPv6 安全隐患**: 1. 庞大的 128 位地址空间让攻击者可轻易更换源地址，降低 IP 过滤和黑名单有效性；2. 若未升级安全监控产品，IPv6 流量将变成不受监控的隐蔽通道。

### ARP 关注点

#### ARP 攻击
*   **ARP 缓存投毒 / ARP 欺骗**: 攻击者通过向目标发送虚假的 ARP 回复，将其伪造的信息插入到系统的 ARP 缓存中。

#### ARP 防御
*   **交换机端口安全**: 防御基于 ARP 攻击的最佳方法，禁止与未知/未授权设备通信，并拦截违规系统发出的 ARP 回复。
*   **arpwatch**: 常用于检测网络中 ARP 投毒攻击的安全工具。

### 安全通信协议

*   **安全通信协议**: 为特定应用通信通道提供安全服务的协议。
*   **互联网协议安全 (IPSec)**: 使用公钥密码学提供加密、完整性、抗重放、访问控制和消息来源身份验证（基于 IP 的协议）。
*   **Kerberos**: 为用户提供单点登录 (SSO) 解决方案，并为登录凭据提供保护。
*   **安全外壳协议 (SSH)**: 端到端加密技术的典型代表，可用于加密众多明文实用程序。
*   **Signal 协议**: 为语音通信、视频会议和文本消息服务提供端到端加密的密码协议。
*   **安全远程过程调用 (S-RPC)**: 用于跨网络服务通信的身份验证服务，防止在远程系统上未经授权执行代码。
*   **传输层安全协议 (TLS)**: 通过加密 TCP 通信的有效载荷在 OSI 第 4 层运行的加密协议。

### 多层协议与融合协议

#### 多层协议概念
*   **TCP/IP 多层协议**: 主要优势之一是支持封装 机制。
*   **分布式网络协议 3 (DNP3)**: 多层开放公共标准协议，主要用于电力和水务公用事业，支持数据采集系统与系统控制设备（变电站计算机、RTU、IED 和 SCADA 主站）之间的通信。

#### 融合协议
*   **融合协议**: 将专用/专有协议与标准协议（如 TCP/IP）合并。优势：允许组织利用现有 TCP/IP 网络基础设施托管特殊服务，无需额外部署专用网络硬件。
*   **存储区域网络 (SAN)**: 独立于主通信网络的辅助网络，将数据存储信号封装/融合到 TCP/IP 通信中，把存储设备整合为网络可访问的综合存储容器。
*   **互联网小型计算机系统接口**: 基于 IP、运行在第 5 层（会话层）的网络存储标准，通过 LAN/WAN/互联网实现与位置无关的文件存储、传输和检索。是光纤通道的低成本替代方案。
*   **以太网上的 InfiniBand (IBoE)**: 将用于高性能计算 (HPC) 环境的低延迟 InfiniBand 流量封装在以太网帧中，允许其直接在现有以太网基础设施上运行。
*   **计算快速链路 (CXL)**: 高级高速互连融合协议技术，支持 CPU、GPU、加速器和内存等在单一高速互连上通信协作，专为 AI 等现代数据密集型工作负载设计。
*   **网络电话**: 基于IP的语音传输。

### VoIP 与 SRTP

#### VoIP (Voice over IP)
- **定义**: 一种隧道机制和融合协议，将音频、视频及其他数据封装到 IP 数据包中，结合应用层封装技术与 TCP/IP 协议栈来支持语音通话和多媒体协作。

#### SRTP (Secure Real-Time Transport Protocol)
- **定义**: 对实时传输协议（RTP）的安全改进。
- **核心机制**: 通过强大的加密和可靠的身份验证技术，将 DoS、中间人攻击及其他 VoIP 漏洞的风险降至最低。

### 软件定义网络与虚拟化

#### SDN (Software-Defined Networking)
- **定义**: 一种独特的网络架构设计，核心是将**基础设施层（数据平面/硬件）**与**控制层（控制平面/管理服务）**分离，使网络可从中心位置直接编程和管理。

#### SDN 接口
- **南向接口**: SDN 控制器到**数据平面网络设备**的通信路径，用于促进网络流量的控制和管理。
- **北向接口**: SDN 控制器到**应用层应用程序/服务**的通信路径，允许应用与控制器交互并利用网络可编程性。
- 💡 **记忆 aid**: **南**下接**地**（数据平面/底层硬件），**北**上接**天**（应用层/上层业务）。

#### NFV (Network Functions Virtualization)
- **定义**: 将网络功能从专用硬件设备中虚拟化和抽象出来。
- **与 SDN 结合**: 允许在控制层中将数据传输路径、通信决策树和流量控制虚拟化，形成灵活且可编程的网络基础设施。

#### 存储虚拟化技术
| 技术 | 全称 | 定义 |
| :--- | :--- | :--- |
| **VSAN** | Virtual SAN | 在虚拟化网络或 SDN 之上虚拟重建 SAN 的软件定义共享存储系统，绕过传统 SAN 对复杂硬件的要求。 |
| **SDS** | Software-Defined Storage | SAN 或 NAS 的 SDN 版本，策略驱动、独立于底层实际物理存储硬件的虚拟存储管理和配置解决方案。 |

#### SD-WAN (Software-Defined Wide-Area Networks)
- **定义**: SDN 的一种演进形式，主要用于管理广域网（WAN）链路上相距遥远的数据中心、远程位置和云服务之间的连接和控制服务。

### 网络分段

#### 网络分段
- **定义**: 将网络细分为较小的组织单元（如子网）。
- **目的**: 提升网络性能（减少拥塞和广播风暴）、提高安全性（隔离流量、限制访问）。

#### 物理分段
- **定义**: 物理上分离网络或系统中的不同组件或网段，以增强安全性、隔离敏感信息并控制访问。
- **类型**: 带内分段、带外分段、物理隔离。

#### 带内与带外分段
- **带内分段**: 数据流量和控制流量使用**相同**的通信路径或网络基础设施。通常等同于逻辑分段。
- **带外分段**: 数据流量和控制流量分离到**不同**的通信路径或网络上，为控制信号和管理流量提供专用网络。
- ⚠️ **易错点**: 带内/带外的核心区分在于**数据流量与控制/管理流量是否分离**，而非单纯的物理隔离。

#### 逻辑分段与虚拟化技术
| 技术 | 全称 | 定义 |
| :--- | :--- | :--- |
| **VLAN** | Virtual Local Area Networks | 将物理局域网逻辑划分为多个隔离的广播域，同 VLAN 设备即使在不同物理网段也可像在同一物理网络上通信。 |
| **VRF** | Virtual Routing and Forwarding | 允许多个路由表实例在同一物理路由器内共存，每个实例作为独立路由域运行，隔离路由信息防泄露。 |
| **虚拟域** | Virtual Domains (虚拟系统/上下文) | 创建网络设备（如防火墙/交换机）的隔离实例，每个拥有自己的配置并作为独立逻辑实体操作。 |
| **VPC** | Virtual Private Cloud | 云服务商提供的虚拟化网络基础设施，允许在公共云中创建和管理隔离的、逻辑分段的网络。 |

#### 微分段与零信任
- **微分段**: 将网络划分为小型的、隔离的网段（甚至小到单个终端设备），对各个工作负载应用细粒度安全控制。
- 💡 **关联**: 微分段是实施**零信任**架构的关键要素。

#### 网络覆盖与分布式架构
- **网络覆盖**: 在现有物理网络之上创建逻辑网络。
- **分布式防火墙 (ISFWs)**: 部署在网络内多个点，更靠近工作负载/设备，允许细粒度自定义安全规则。
- **分布式路由器**: 部署在网络内各个点而非集中部署，实现本地化路由决策，高效控制微分段间流量。

#### VXLAN (Virtual Extensible LAN)
- **定义**: 定义在 RFC 7348 中的封装协议。
- **功能**: 允许 VLAN 延伸跨越子网和地理距离。
- ⚠️ **考点**: VXLAN 支持创建多达 **1600 万**个虚拟网络（极大突破了传统 VLAN 的 4096 个限制）。

### 边缘网络

#### 边缘网络
- **定义**: 战略性将计算资源分配给网络内边缘设备的数据架构。
- **目的**: 将处理需求从中央服务器分散，将内容/服务/应用部署到离用户更近的地方，减少延迟，提升性能。

#### 边缘网络对等互联
- **定义**: 在不同边缘网络之间建立直接互连，无需依赖中间媒介即可直接交换网络流量。
- **目的**: 优化数据交换，实现性能提升、延迟降低和效率增强。

### 无线网络

#### 无线基础概念
- **802.11**: 用于无线网络通信的 IEEE 标准。
- **自组织模式**: 无需集中控制授权（基站或接入点），任何两个无线设备可直接通信。
- **SSID**: 分配给无线网络以区分不同网络的标识，即 Wi-Fi 网络名称。
- **信标帧**: WAP 广播 SSID 所使用的特殊传输，使覆盖范围内无线网卡可见该网络。

#### 无线部署评估
- **现场调查**: 使用 RF 信号检测器对信号强度、质量和干扰进行正式评估。
- **热图**: 在建筑物蓝图上测量信号强度的映射，定位热点（过饱和）和冷点（缺乏），指导 WAP 放置。

#### 无线安全协议演进
| 协议 | 加密/认证机制 | 特点/缺陷 |
| :--- | :--- | :--- |
| **WEP** | 静态共享 RC4 密钥 / SKA | ⚠️ **极度脆弱**：因实现缺陷，已被淘汰。 |
| **WPA** | RC4 + TKIP / LEAP | WEP 替代方案；与主机协商唯一密钥集；分离了认证与加密。 |
| **WPA2** | **AES-CCMP** | 取代 WEP/WPA (IEEE 802.11i)，用 AES 替代 RC4。 |
| **WPA3** | AES-CCMP + **SAE** | 个人版采用 SAE 取代预共享密钥(PSK)认证。 |

#### 核心加密与认证技术
- **CCMP**: 结合两种块密码模式使块算法支持流处理；AES-CCMP 是 WPA2 和 WPA3 **首选**的无线加密手段。
- **SAE (对等同时身份验证)**: WPA3-PER 使用。使用密码但不通过连接发送，而是执行**蜻蜓密钥交换** 的零知识证明过程进行认证和会话密钥交换。
- **802.1X/EAP**: 基于端口的网络访问控制标准，作为切换系统利用现有认证服务（如 RADIUS 或 TACACS+）。
- **EAP**: 不是一个具体的认证机制，而是一个**认证框架**，允许新认证技术与现有技术兼容。
- **PEAP**: 将 EAP 方法封装在 **TLS 隧道**中，为通常不加密的 EAP 提供认证和潜在的加密保护。
- **WPS**: 简化添加客户端，可通过按钮或 PIN 码触发。⚠️ **安全陷阱**：PIN 码极易受暴力破解攻击。

#### 天线类型
- **全向天线**: 直列或杆式天线（基站天线/橡胶鸭天线），360度覆盖，常见于基站和客户端。
- **定向天线**: 将收发能力集中在单一方向。示例：八木、罐状、面板、抛物面天线。

#### 无线传输技术
- **扩频**: 通信在多个频率上发生，消息分块同时使用不同频率并行发送。
- **OFDM**: 数字多载波调制，使用相互垂直（正交）且互不干扰的信号，实现更紧凑传输和更高吞吐量。
- **MIMO**: 发射端和接收端使用多个天线，提高性能、数据速率、可靠性和频谱效率。

#### 短距离与无线跟踪技术
| 技术 | 特点 |
| :--- | :--- |
| **蓝牙** | 2.4 GHz，短距离，创建个人局域网 (PAN)。 |
| **BLE** | 低功耗衍生版，专为 IoT/移动/医疗设计。⚠️ **注意**：与标准蓝牙**不兼容**，但可在同一设备上共存。 |
| **RFID** | 跟踪技术，磁场感应供电，可从相当远的距离触发并读取标签代码。 |
| **NFC** | 源自 RFID，极近距离（≤4 厘米），用于自动同步、数据交换或非接触式支付。 |

#### 无线攻击向量
- **战争驾驶**: 使用检测工具（手持设备、笔记本、无人机等）寻找未授权访问的无线网络信号。
- **恶意接入点**: 未经授权安装/操作的 WAP，缺乏安全配置，成为不受监管的访问路径。
- **邪恶双胞胎**: 攻击者操作虚假 AP，欺骗/克隆原始基站的 MAC 地址和 SSID，诱骗客户端连接以进行中间人攻击。
- **解除关联攻击**: 发送伪造的无线管理帧断开客户端与 WAP 的连接，可导致 DoS、会话劫持或引发中间人攻击。
- **阻塞**: （文档截断，无详细定义，通常指发射干扰信号破坏无线通信）

### 无线与网络攻击

#### 阻塞
*   **定义**：通过传输无线电信号来降低有效信噪比，从而故意阻止或干扰通信的攻击行为。

#### 初始化向量（IV）滥用
*   **定义**：当密码功能中使用的随机数（即初始化向量）太短、以明文形式交换或选择不当时，成为一个弱点被攻击者利用。
*   ⚠️ **经典案例**：WEP加密破解即利用了IV的弱点。

#### 重放攻击
*   **定义**：涉及重新传输捕获的通信流量（如连接或身份验证请求），以试图在无需掌握实际凭据的情况下获得目标系统的访问权限或导致拒绝服务。

#### 蓝牙攻击分类
*   **蓝劫**：未经所有者或用户许可，向具有蓝牙功能的设备发送未经请求的消息。
*   **蓝窃**：通过蓝牙连接未经授权访问数据的行为（通常发生在已配对链路上，或已知目标MAC地址时针对不可发现设备进行）。
*   **蓝虫**：通过蓝牙连接远程控制设备的硬件和软件（例如启用受损系统上的麦克风将其用作远程无线窃听器）。
*   💡 **记忆类比**：**Jack**ing = 垃圾短信**塞**入；**Snarf**ing = 数据**窃**取；**Bug**ging = 设备被**监听/控制**。

### 卫星与蜂窝网络通信

#### 卫星通信
*   **定义**：基于在地面位置和轨道人造卫星之间传输无线电波进行通信。用于支持电话、电视、无线电、互联网和军事通信。

#### 蜂窝网络
*   **定义**：围绕被称为“小区”的接入区域组织的移动/无线网络，这些小区以主收发器（蜂窝基站或信号塔）为中心。

#### 蜂窝网络代际架构
*   **4G 通信架构**：主要采用基于IP的通信技术处理语音和数据，取代传统电路交换电话服务。最常见传输系统是LTE，其次是WiMAX。
*   **5G 传输**：使用比以往更高的频率，传输速度更高，但代价是传输距离缩短。

#### 蜂窝网络攻击与漏洞
*   **拦截与伪造攻击**：无线传输可被特定无线嗅探设备拦截；攻击者可模拟服务商信号塔实施中间人攻击。
*   **设备桥接漏洞**：利用蜂窝电话连接访问企业网络会提供潜在攻击途径，设备可能充当网桥，在公司内部网络中创建不安全的访问节点。

### 安全网络组件与架构

#### 网络区域划分
*   **内部网**：私密托管信息服务的专用网络，仅内部用户可访问Web、邮件等服务，外部人员无法访问。
*   **外部网**：互联网和内部网的交叉点，向授权的外部人员或实体提供信息的网络划分。
*   **屏蔽子网**：以前称为非军事区（DMZ），是特殊用途的外部网，专供信任度低和未知的用户访问特定系统（如公众Web服务器）。
*   **屏蔽主机**：逻辑上位于网段内部受防火墙保护的系统。所有入站流量路由至该主机，它充当受信任系统的代理，过滤流量并保护内部系统身份。

#### 域与网络设备
*   **冲突域**：一组联网系统，若其中任何两个或多个系统同时传输数据，就会引起冲突。
*   **广播域**：一组联网系统，当一个成员发送广播信号时，所有其他成员都会接收到。
*   **网桥**：连接两个网络（甚至不同拓扑、布线类型和速度），连接使用相同协议的网段。在OSI第2层运行。
*   **交换机**：通过MAC地址管理帧传输。主要在OSI第2层运行；具有VLAN路由等附加功能的可在OSI第3层运行（第3层交换机）。
*   **路由器**：基于逻辑IP寻址管理流量，控制网络流量，连接相似网络。在OSI第3层运行。
*   ⚠️ **易混点**：网桥/交换机工作在第2层（MAC），路由器工作在第3层（IP）。

#### 其他关键组件
*   **跳板服务器**：部署在外部网、屏蔽子网或云网络中的远程访问系统，旨在使访问特定系统或网络更安全便捷。
*   **系统级芯片**：将计算机所有元素（CPU、RAM、GPU、Wi-Fi等）集成到一个芯片中的集成电路，通常仅缺少大容量存储。
*   **网络访问控制 (NAC)**：通过严格执行安全策略控制环境访问。自动检测响应，确保补丁最新、配置合规，将未授权设备挡在网络外。
*   **端点检测与响应 (EDR)**：检测、记录、评估和响应由问题软件或用户引起的可疑活动和事件的安全机制。

### 防火墙与代理技术

#### 防火墙类型
*   **防火墙**：部署在信任度较高和较低区域之间，根据规则（过滤器或ACL）过滤流量的硬/软件。
*   **静态包过滤防火墙**：检查消息头数据过滤流量，关注源/目标IP地址（第3层）和端口号（第4层）。
*   **应用层防火墙**：基于单一的互联网服务、协议或应用程序过滤流量，在OSI第7层运行。
*   **状态检测防火墙**：评估网络流量的状态、会话或上下文。了解出站通信会触发外部响应，自动创建仅在对话存在时的临时响应规则。
*   💡 **记忆对比**：静态包过滤 = 只看信封（IP/端口）；应用层 = 拆信看内容（第7层）；状态检测 = 记住之前的对话上下文。

#### 代理服务
*   **正向代理**：充当查询外部资源的中间人，处理内部客户端访问外部服务的查询。
*   **反向代理**：处理外部系统对内部托管服务的入站请求。
*   💡 **记忆类比**：正向代理 = 内部员工的**出国代购**；反向代理 = 公司的**前台接待**（挡住外部访客，转交内部）。

### 网络拓扑与传输技术

#### 网络拓扑
| 拓扑类型 | 特征 | 单点故障 (SPOF) |
| :--- | :--- | :--- |
| **环形** | 系统连成圆，单向传输循环，令牌传递控制流量 | 无（除非环断裂） |
| **总线** | 系统连到主干电缆，单网段故障不影响其他网段 | 中央主干线 |
| **星形** | 集中连接设备（集线器/交换机），专用网段连接 | 中央连接设备 |
| **网状** | 多条路径互连，提供冗余，多网段故障不影响整体 | 无（高冗余） |

#### 传输技术
*   **基带技术**：仅支持单个通信信道，数字信号形式（如以太网）。
*   **宽带技术**：支持多个同时信号，使用频率调制支持多信道，模拟信号形式。

#### 介质访问控制
*   **CSMA/CD (冲突检测)**：主机传输时侦听冲突，若检测到冲突则发送拥塞信号，所有主机停止并等待随机时间后重试。
*   **CSMA/CA (冲突避免)**：通过在任何给定时间仅授予单个通信权限来避免冲突（如802.11无线网络）。
*   ⚠️ **区分考点**：CSMA/**C**D 侧重**检测**（有线以太网常用）；CSMA/**C**A 侧重**避免**（无线网络常用，因为无线环境下冲突检测困难）。
