# 第12章 安全通信与网络攻击

### 12.1 协议安全机制

#### 点对点协议 (PPP)
*   **定义**：一种封装协议，支持通过拨号或点对点链路传输IP流量。
*   **特点**：取代了SLIP协议；支持自动连接配置、错误检测、全双工通信；支持多种身份验证选项（PAP, CHAP, EAP）。

#### 身份验证协议对比

| 协议名称 | 缩写 | 核心机制与特点 | 安全性评估 |
| :--- | :--- | :--- | :--- |
| **密码身份验证协议** | PAP | 明文传输用户名和密码，仅提供凭据传输方法 | ⚠️ 极不安全，无加密 |
| **质询握手身份验证协议** | CHAP | 使用不可重放的质询-响应对话；定期重新验证 | ⚠️ 基于MD5，已不再被认为是安全的 |
| **MS-CHAPv2** | MS-CHAPv2 | 微软对CHAP的定制版，使用更新算法，添加可选会话加密 | 比CHAP更安全，但仍有已知漏洞 |

#### 可扩展身份验证协议 (EAP) 家族
*   **EAP (可扩展身份验证协议)**：是一个**身份验证框架**而非具体协议，允许定制身份验证安全解决方案（如支持智能卡、令牌、生物识别）。

| EAP 变体 | 全称 | 核心特点 |
| :--- | :--- | :--- |
| **LEAP** | 轻量级EAP | 思科专有，WPA TKIP替代方案（在802.11i/WPA2标准前开发），现已过时 |
| **EAP-FAST** | 安全隧道灵活身份验证 | 思科提出，用于取代过时的LEAP |
| **PEAP** | 受保护的可扩展身份验证协议 | 将EAP封装在TLS隧道中；支持双向验证；因强制自身安全性比单独EAP更受青睐 |
| **EAP-TLS** | EAP传输层安全 | 开放IETF标准，实现TLS保护验证流量；客户端和服务器都拥有数字证书（双向证书）时最有效 |
| **EAP-SIM** | 用户身份模块 | 通过GSM网络对移动设备进行验证，使用与账户/服务关联的SIM卡 |
| **EAP-MD5** | EAP-MD5 | 最早的EAP方法之一，使用MD5哈希密码 | ⚠️ 已被弃用 |
| **EAP-POTP** | EAP受保护的一次性密码 | 支持MFA中使用OTP令牌（硬/软件），可用于单向和双向验证 |

#### 网络访问与端口安全
*   **IEEE 802.1X (基于端口的网络访问控制)**：使用封装的EAP确保客户端在完成验证前无法通信。充当验证代理，将无线客户端请求转发给AAA服务器。
*   **物理端口安全**：对物理连接点（如RJ-45插孔/设备端口）的控制，防止未授权物理连接。
*   **TCP/UDP端口安全**：利用防火墙、IDS/IPS管理端口状态，检测并阻止端口扫描活动。

#### 网络性能与质量指标

| 概念 | 缩写 | 定义 |
| :--- | :--- | :--- |
| **服务质量** | QoS | 对网络通信效率和性能的监督与管理，保护网络在负载下的可用性 |
| **延迟** | - | 数据包从源传输到目的地所需的时间 |
| **抖动** | - | 不同数据包之间延迟的变化差异 |
| **丢包** | - | 数据包在传输中丢失需重传的现象 |
| **干扰** | - | 电噪声、设备故障等破坏数据包内容的因素 |
| **吞吐量** | - | 给定时间内成功传输的实际数据量，代表真实世界性能 |
| **信噪比** | SNR | 信号强度与背景噪声/干扰水平的比较 |
| **透明度** | - | 安全控制不被用户察觉的特性；越透明，用户绕过或意识到的可能性越小 |

> 💡 **记忆 aid (QoS指标)**：想象开车送货——**延迟**是全程耗时，**抖动**是路况导致的时快时慢，**丢包**是货物掉落需重送，**吞吐量**是实际送达的货物总量。

### 12.2 语音通信的安全

*   **公共交换电话网 (PSTN / POTS)**：传统电话服务，常在宽带/VoIP故障时作为备用线路。
*   **互联网协议语音 (VoIP)**：将音频封装到IP数据包中通过TCP/IP网络通话，也是多媒体消息服务的基础。
*   **语音钓鱼**：基于语音的社会工程攻击。⚠️ 攻击者常利用VoIP伪造来电显示掩盖身份。
*   **电话飞客**：专门针对电话系统（PSTN/移动/PBX/VoIP）的攻击，绕过机制以盗打免费长途、窃取服务或造成中断。
*   **专用交换机 (PBX)**：私有组织的电话交换系统，多内部终端共享少量外部PSTN线路。
*   **直接向内系统访问 (DISA)**：为减少外部PBX欺诈，对所有连至PBX的外部连接强制增加身份验证要求的技术。

### 12.4 多媒体协作

*   **多媒体协作**：使用支持多媒体的通信方案增强远程协作。

### 12.5 监控和管理

*   **网络可观测性**：通过收集分析指标、日志和追踪，获取网络内部状态洞察的能力。
*   **容量管理**：规划、监控和优化网络容量，确保处理当前/未来需求，防性能下降并保可扩展性。
*   **故障检测与处理**：识别和解决网络问题，尽早发现以减少停机，实施容错和弹性策略。

### 12.6 负载均衡

*   **负载均衡**：将网络流量分散到多个链路/设备，以优化利用率、最小化响应时间、最大化吞吐量、减少超载并消除瓶颈。

#### 负载均衡调度技术

| 调度技术 | 分配逻辑 |
| :--- | :--- |
| **随机选择** | 随机分配到目的地服务器 |
| **轮询** | 按固定顺序依次分配 (1, 2, 3, 1, 2, 3...) |
| **负载监控** | 分配给当前负载最低的设备/路径 |
| **偏好/加权** | 基于主观偏好或已知容量差异（如按容量比例分配） |
| **最少连接/流量/延迟** | 分配给活动连接最少、流量最低或延迟最小的目的地 |
| **基于位置-地理** | 根据地理距离分配（常用于集群分散或跨多跳路由） |
| **基于位置-亲和性/持续性** | 基于同一客户端先前连接记录分配，确保后续请求去往相同目的地（优化连续性） |

#### 负载均衡架构概念

*   **虚拟IP地址 (VIP)**：映射到服务器集群而非特定物理接口的IP，作为流量入口点，确保负载均匀分布并防单点瓶颈。
*   **全局服务器负载均衡 (GSLB)**：跨全球多数据中心/地理位置分配流量，考虑距离/服务器健康等因素优化全球性能。
*   **主动-主动系统**：正常操作时利用所有可用路径/系统以最大化处理能力。⚠️ 故障时，剩余活动路径必须承担全部负载（存在过载风险）。

### 电子邮件安全

#### 关键概念
*   **可接受的使用策略**: 明确定义在组织的电子邮件基础设施上允许和禁止的活动。
*   **邮件炸弹**: 通过向系统发送大量消息来淹没系统的一种拒绝服务 攻击。
*   **邮件风暴**: 当有人对收件人和抄送人众多的消息使用“全部回复”时产生的 DoS 安全问题。
*   **垃圾邮件**: 发送不需要的、不适当的或不相关消息的行为。
*   **安全多用途互联网邮件扩展 (S/MIME)**: 电子邮件安全标准，通过公钥加密、数字信封和数字签名提供身份验证和机密性。
*   **良好隐私密码法 (PGP)**: 基于点对点公私钥的电子邮件系统，使用多种加密算法加密文件和电子邮件消息。
*   **域名密钥识别邮件 (DKIM)**: 身份验证方法，允许接收方邮件服务器检查宣称来自特定域的邮件是否确实经过该域所有者授权。
*   **发件人策略框架 (SPF)**: 通过检查入站消息是否源自被 SMTP 原始域所有者授权发送消息的主机来运作，防止垃圾邮件和邮件欺骗。
*   **STARTTLS**: 一个 SMTP 命令（非协议），用于在目标系统支持 TLS 的情况下协商建立加密通道以实现安全连接。
*   **隐式安全简单邮件传输协议**: SMTP 的 TLS 加密形式，假定目标服务器支持 TLS，并针对 TCP 465 端口发起通信。
*   **传真加密器**: 赋予传真机使用加密协议打乱传出传真信号能力的设备。

> 💡 **记忆 aid (邮件认证三剑客)**: **SPF** 验证**发件服务器**是否合法（防伪造源）；**DKIM** 验证**邮件内容**是否被篡改且确属域所有者（数字签名）；**DMARC**（虽未在此处详述，常与两者结合）规定如何处理未通过 SPF/DKIM 的邮件。

> ⚠️ **考试陷阱**: **STARTTLS** 不是协议，而是 **SMTP 命令**！它将明文连接升级为加密连接；而 **Implicit SMTPS** 直接假定对方支持 TLS 并使用 TCP 465 端口。

### 虚拟专用网络 (VPN)

#### 关键概念
*   **虚拟专用网络 (VPN)**: 在两个实体之间跨越中间不可信网络建立的通信通道，提供访问控制、身份验证、机密性和完整性。
*   **VPN 集中器**: 专用硬件设备，支持大量（数百或数千个）并发 VPN 连接。
*   **隧道技术**: 将一种协议封装在另一种协议的数据包中以保护其内容的网络通信过程。
*   **始终在线 VPN**: 每次网络链路激活时自动尝试连接 VPN 服务的配置，降低使用开放公共网络的风险。

#### VPN 模式
| 模式 | 终止位置 | 加密范围 | 原始消息头 |
| :--- | :--- | :--- | :--- |
| **传输模式** | 各个主机 | 仅有效载荷 | 保持完整无缺 |
| **隧道模式** | 网络边界的 VPN 设备 | 有效载荷 + 消息头 | 被封装并添加临时头部 |

#### VPN 隧道配置
| 配置类型 | 流量路径 | 特点 |
| :--- | :--- | :--- |
| **分割隧道** | 组织网络流量走 VPN，互联网流量直连 | 允许同时访问，但互联网流量不受组织安全设施保护 |
| **全隧道** | 所有流量均通过 VPN 发至组织网络，再路由至互联网 | 所有流量均受组织安全基础设施过滤和管理 |

#### VPN 协议
*   **点对点隧道协议 (PPTP)**: 过时的封装协议，运行在 OSI 第 2 层，使用 TCP 1723 端口。初始隧道协商过程不加密，易被拦截。
*   **第 2 层隧道协议 (L2TP)**: 结合 PPTP 和思科 L2F 特性，运行在第 2 层，使用 UDP 1701 端口。不提供原生加密，常与 IPSec ESP 结合使用。
*   **通用路由封装 (GRE)**: 思科专有隧道协议，提供封装但**不提供加密**。
*   **OpenVPN**: 基于 TLS 的开源 VPN 实现，易配置且稳健，使用预共享密码或证书进行身份验证。
*   **IP 安全协议**: IP 安全扩展标准（IPv4 附加，IPv6 集成），用于建立 VPN 链接，提供身份验证和加密。

> ⚠️ **考试陷阱**: **PPTP** 初始协商**不加密**；**L2TP** 本身**无加密**（需配合 IPSec）；**GRE** 只有封装**无加密**。只有 **IPSec** 和 **OpenVPN** 提供完整的加密与安全功能。

#### IPSec 协议族
*   **身份验证头**: 提供消息完整性和不可否认性，实施主要身份验证功能、会话访问控制，防止重放攻击。
*   **封装安全有效载荷**: 提供有效载荷的机密性（加密）和完整性，提供有限身份验证，防重放攻击，支持传输/隧道模式。
*   **IP 有效载荷压缩**: 在 ESP 加密前使用的压缩工具，旨在保持线速传输。
*   **互联网密钥交换**: 管理加密密钥的机制，由以下三个元素组成：
    *   **OAKLEY**: 类似 Diffie-Hellman 的密钥生成和交换协议。
    *   **安全密钥交换机制 (SKEME)**: 类似数字信封的安全交换密钥方法。
    *   **互联网安全关联和密钥管理协议 (ISAKMP)**: 组织和管理 OAKLEY 和 SKEME 生成的密钥，协商并为安全关联提供经过身份验证的密钥材料。

> 💡 **记忆 aid (IPSec 核心)**: **AH** 防**篡改**（完整性/认证），**ESP** 防**窃听**（机密性/加密）。IKE 负责密钥管理（OAKLEY生密钥，SKEME传密钥，ISAKMP管密钥）。

### 交换与虚拟局域网

#### 关键概念
*   **交换机**: 现代网络管理设备，主要在 OSI 第 2 层运行，核心功能：学习、转发、丢弃、泛洪。
*   **内容寻址存储器表 (CAM Table)**: 存储在交换机内存中的表，包含 MAC 地址与交换机物理端口号的映射关系。
*   **虚拟局域网 (VLAN)**: 基于硬件的网络分段技术，在逻辑上分割网络而不改变物理拓扑。
*   **私有 VLAN / 端口隔离**: 成员只能彼此交互及通过预定的专用上行链路端口通信的 VLAN。
*   **中继端口**: 专用交换机端口，高带宽，用于连接多台交换机。
*   **VLAN 标签 (VLAN Tagging)**: 基于 IEEE 802.1q，修改以太网帧头包含 VLAN 标签值，使 VLAN 可跨物理交换机延伸。
*   **网络虚拟化**: 软硬件结合为单一实体，通过软件控制台统一管理网络功能（管理、流量整形、地址分配等）。

#### 监控与安全机制
*   **端口镜像**: 将一个或多个端口流量复制到审计/IDS/监控端口，用于流量分析和入侵检测。
*   **端口分流器**: 物理内联设备，当端口镜像不可用或无法满足需求时用于窃听网络通信。

#### MAC 地址相关攻击与防御
*   **MAC 泛洪攻击**: 发送大量随机源 MAC 帧填满 CAM 表，迫使交换机回退到泛洪模式以窃听通信。
*   **MAC 欺骗**: 更改系统软件中的默认 MAC 地址为其他值。
*   **MAC 克隆**: 冒充已授权网络设备的 MAC 地址，以绕过端口安全或 MAC 过滤。
*   **MAC 过滤**: 限制仅具有特定已知 MAC 地址的设备访问网络的安全机制。

> ⚠️ **考试陷阱**: **VLAN 跳跃** 攻击是利用**双重标签**的帧欺骗旧式交换机，使帧跨 VLAN 传输。MAC 泛洪的原理是耗尽 **CAM 表**，导致交换机降级为 **Hub (泛洪模式)**。

### 网络地址转换 (NAT)

#### 关键概念
*   **网络地址转换 (NAT)**: 隐藏内部客户端 IPv4 配置，将私有 IP 替换为公共外部 IP。
*   **端口地址转换 (PAT / NAT 重载 / NAPT)**: 使用传输层端口号将内部套接字映射到外部套接字，允许单个公共 IPv4 托管多达 65,536 个内部通信会话。
*   **NAT 穿透 (NAT-T)**: 专为 IPSec 等隧道协议设计，解决传统 NAT 更改数据包头部导致 IPSec 完整性受损的问题，允许 NAT 和 VPN 在同一设备共存。
*   **静态 NAT (Static NAT / DNAT / Port Forwarding / Reverse Proxy)**: 入站映射技术，允许外部实体通过映射到内部私有地址的公共套接字发起通信。
*   **IPv6 网络地址转换 (NAT66)**: 将多个私有 IPv6 映射到较小公共 IPv6 池的技术。

#### 特殊 IP 地址
*   **私有 IPv4 地址 (RFC 1918)**: 默认不被路由到互联网，保留范围如下：

| 类别 | 私有 IP 范围 | 
| :--- | :--- |
| **A 类** | 10.0.0.0 - 10.255.255.255 |
| **B 类** | 172.16.0.0 - 172.31.255.255 |
| **C 类** | 192.168.0.0 - 192.168.255.255 |

*   **自动私有 IP 分配 (APIPA)**: Windows 系统在 DHCP 失败时自动分配的 IP 范围：**169.254.0.1 - 169.254.255.254**，默认子网掩码：**255.255.0.0**。
*   **环回地址**: 纯软件接口，用于测试本地网络设置。保留整个 127.x.x.x 范围，最常用 **127.0.0.1**。

> 💡 **记忆 aid (APIPA)**: 看到 **169.254.x.x** 就想到 DHCP 服务器挂了（Automatic Private IP Addressing = 电脑自己给自己找后路）。

### 第三方连接

#### 关键概念
*   **第三方连接**: 组织 IT 环境直接连接另一实体网络，导致本地威胁和风险相互影响，一方的妥协易导致另一方妥协。
*   **谅解备忘录/协议备忘录 (MOU/MOA)**: 表达两个实体意向、意愿或目的一致的文件。**不具有法律约束力**，仅记录协议或安排的具体细节（意向书）。
*   **互联安全协议**: 对两个组织 IT 基础设施间链路的安全立场、风险和技术要求的**正式声明**，界定维护通信路径安全的期望和责任。

> ⚠️ **考试陷阱**: **MOU/MOA** 是**非法律约束**的意向文件；**ISA** 是针对安全和技术要求的**正式声明**（通常具有强制约束力）。考试常考两者的法律效力区别。

### 云访问安全代理与广域网技术

#### 云访问安全代理 (CASBs)
**Cloud Access Security Brokers (CASBs)**
*   **定义：** 在组织采用云服务时部署的技术，用于强制执行已建立的安全要求、指南和策略。

#### 广域网技术 (WAN Technologies)
**WAN Technologies**
*   **定义：** 用于在广阔的地理区域内建立连接的关键技术，通常涉及电信提供商（提供专线、MPLS、VPN等服务）和硬件支持以确保连续的连接。

| WAN 技术类型 | 全称 | 定义与特点 |
| :--- | :--- | :--- |
| **专线** | Leased lines | 在广域网连接中提供专用的点对点连接的服务。 |
| **多协议标签交换** | Multiprotocol Label Switching (MPLS) | 使企业能够跨越多个位置建立私有和安全网络的服务；在共享基础设施上实现安全高效的数据传输；结合了服务质量功能以满足特定性能需求。 |
| **软件定义广域网** | Software-defined WAN (SD-WAN) | 利用软件定义网络原理的现代广域网连接方法；允许组织在各种连接上动态路由流量，从而优化性能和成本效益。 |

### 交换技术 (Switching Technologies)

#### 交换技术概述
*   **定义：** 为了简化在通过多个中间网络连接的两个系统（单台计算机或局域网）之间传输数据的复杂过程而开发的技术。

#### 电路交换 (Circuit Switching)
*   **定义：** 在通信双方之间建立一条专用的物理路径，该路径在整个通信会话期间被当前通信伙伴独占使用且保持不变。
*   **特点：** 提供固定的传输时间、统一的质量水平，极大地减少信号丢失或通信中断。

#### 分组交换 (Packet Switching)
*   **定义：** 将通信消息分解为小段数据（分组或信元），每个数据段拥有包含源和目的信息的独立报头以供中间系统进行路由。
*   **特点：** 通信通道仅在数据包实际传输时被短暂占用，发送完毕后即刻释放，不强制通信路径的独占性。
*   **安全与可靠性：**
    *   *风险：* 同一物理连接承载不同来源的数据，存在信息泄露或被窃听风险，需通过加密和流量隔离保护。
    *   *优势：* 不高度依赖特定物理连接，物理路径损坏时可使用备用路径，比电路交换具备更好的抗中断能力。

#### 虚拟电路 (Virtual Circuits)
*   **定义：** 在分组交换网络上，于两个特定端点之间创建的逻辑路径或电路。

| 虚拟电路类型 | 全称 | 定义与特点 |
| :--- | :--- | :--- |
| **永久虚拟电路** | Permanent Virtual Circuits (PVCs) | 始终可用的预定义虚拟电路；类似专用租用线路；不使用时可关闭，需要时能被自动且即时地重新打开。 |
| **交换虚拟电路** | Switched Virtual Circuits (SVCs) | 每次通信前利用当前可用最佳路径临时创建；数据传输完成后予以拆除。 |

> 💡 **记忆对比：电路交换 vs 分组交换**
> *   **电路交换** = "专属高铁轨道"（独占物理路径，质量稳定，但资源占用大，断线即中断）。
> *   **分组交换** = "快递物流"（拆包发货，共享路线，灵活抗中断，但需防丢包/偷看）。
> *   **PVC** = "包月专线"（随时待命，开即用）；**SVC** = "按需打车"（用完即拆）。

### 广域网链路与光纤链路

#### 广域网链路 (WAN links)
*   **定义：** 用于将遥远的网络、节点或各个设备连接在一起。
*   ⚠️ **安全陷阱：** 为确保连接安全，特别是在公共网络链路上，必须进行适当的连接管理和传输加密。

#### 数字用户线路 (DSL)
*   **Digital Subscriber Line**
*   **定义：** 利用升级后的电话网络为消费者提供 144 Kbps 至 100 Mbps（或更高）速度的非专用线路技术。
*   **特点：** 存在多种格式，根据提供的具体下行和上行带宽而变化。

#### 综合业务数字网 (ISDN)
*   **Integrated Services Digital Network**
*   **定义：** 原计划作为公共交换电话网（PSTN）的替代品。
*   ⚠️ **现状/边缘考点：** 由于 DSL、有线互联网和光纤等技术的出现未能得到广泛采用，目前大多数 ISDN 服务已停止。

#### 同步数字体系与同步光网络 (SDH and SONET)
*   **定义：** 主要运行在硬件或物理层的光纤高速网络标准，用于规定基础设施和线路速度要求。
*   **标准化组织：** SDH 由国际电信联盟（ITU）标准化；SONET 由美国国家标准学会（ANSI）标准化。
*   **同步时分复用 (Synchronous TDM)：** SDH和SONET使用同步时分复用技术进行高速双工通信，以最大限度减少对控制和管理开销的需求。
*   **同步传输模块 (STM - Synchronous Transport Module)：** 与SDH标准相关联的层级级别，用于在数字信号层次结构中进行组织和复用。

> 💡 **记忆对比：SDH vs SONET**
> *   **SDH** = **国际**标准 (ITU)
> *   **SONET** = **美国**标准 (ANSI)

### 预防或减轻网络攻击

#### 常见通信系统安全威胁
*   **威胁列表：** 拒绝服务、身份伪造、重放攻击、ARP 欺骗、DNS 欺骗、窃听以及传输篡改。

#### 被动攻击
*   **定义：** 不直接干预或修改数据流的攻击行为（如窃听设备和软件）。
*   ⚠️ **检测难点：** 由于不修改数据，通常很难被系统检测到。

#### 主动攻击
*   **定义：** 攻击者将单纯的窃听或搭线窃听行为转变为篡改或向通信流量中注入数据。

> 💡 **记忆对比：被动攻击 vs 主动攻击**
> *   **被动攻击** = "只看不动"（窃听/嗅探），难发现，需靠加密防范。
> *   **主动攻击** = "又看又动"（篡改/注入），易发现，需靠完整性校验防范。

#### 防御对策

| 攻击类型 | 防御对策 |
| :--- | :--- |
| **窃听防御** | 1. 维持物理访问安全<br>2. 对通信使用加密技术（如 IPSec 或 SSH）<br>3. 采用一次性身份验证方法（如一次性密码本或令牌设备）<br>4. 使用应用程序白名单阻止嗅探器等未授权软件运行 |
| **篡改/重放攻击防御** | 1. 使用数字签名验证机制<br>2. 数据包校验和验证（即数据完整性检查） |
