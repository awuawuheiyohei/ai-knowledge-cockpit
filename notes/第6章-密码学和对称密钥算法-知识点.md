# 第6章 密码学和对称密钥算法

## 6.1 密码学基础

### 密码学核心目标
- **保密性**
- **完整性**
- **身份验证**
- **不可否认性**

### 密码系统类型
| 概念 | 定义 |
|---|---|
| **对称密码系统** | 所有通信用户使用同一个共享的秘密密钥进行加密和解密。 |
| **非对称密码系统** | 每个用户拥有独立的公钥和私钥对，公钥加密的消息只能由对应私钥解密，反之亦然。 |

### 密码学基本要素
| 概念 | 符号 | 定义 |
|---|---|---|
| **明文** | P | 消息被转换为密码形式之前的原始状态。 |
| **密文** | C | 使用密码算法对明文消息加密后产生的输出。 |
| **密钥空间** | - | 特定加密算法中所有有效密钥值的范围，由密钥比特大小决定（范围从 0 到 $2^n$，n为密钥位数）。 |
| **算法** | - | 基于数学的规则集，规定加密和解密过程如何进行。 |
| **密码变量** | - | 密码学密钥的另一种称呼（常用于美国政府应用）。 |

### 密码学核心原则与学科
- **柯克霍夫原则**：密码系统应当是安全的，即使除密钥外，关于系统的一切都是公开的。
- **密码学**：创建和实现密码及密文的科学与艺术。
- **密码分析学**：研究如何破解或击败代码和密码的方法与科学。
- **密码学总称**：密码学和密码分析学两门学科的统称。
- **密码模块系统**：代码或密码在硬件和软件中的具体实现。

### 逻辑与数学运算
| 运算类型 | 规则 |
|---|---|
| **逻辑与 (AND)** | 仅当两个输入值都为真(TRUE)时，输出才为真。 |
| **逻辑或 (OR)** | 至少有一个输入值为真(TRUE)，输出就为真。 |
| **逻辑非 (NOT)** | 反转单个输入变量的值。 |
| **异或 (XOR)** | 仅当只有一个输入值为真时返回真；两输入同真或同假返回假。密码学中极常用。 |
| **模运算** | 结果为执行除法运算后保留下来的余数值。 |

### 密码学高级概念
- **单向函数**：极易计算出输出值，但已知输出值几乎不可能反向推导出输入值的数学运算。
- **随机数/临时数**：一次性生成的随机数字，作为占位符变量使用，每次必须唯一。
- **初始化向量 (IV)**：与块大小相同长度的随机位串，与消息进行XOR运算，确保相同密钥加密相同消息时生成独特密文。
- **零知识证明**：向第三方证明自己掌握某一事实（如密码），同时绝不泄露该事实本身的机制。
- **分离知识**：将执行操作所需的信息或权限分散给多个用户，确保无单人拥有破坏安全的特权。
- **密钥托管**：将加密密钥交由第三方（恢复代理）保管，满足特定条件时可使用托管密钥恢复权限或解密。
- **M/N 控制 (M of N Control)**：总数为N的代理中，至少需要M个协同合作才能执行高安全级别任务。
- **工作函数/工作因子**：测量执行完整暴力破解攻击所需的时间和/或成本，以评估密码系统保护强度。

### 古典密码技术
- **代码**：代表单词或短语的符号密码系统，作用于单词/短语级别，目的不一定为机密性。
- **加密**：通过改变或重排字符/位/数据块隐藏消息真实含义的技术。
- **置换密码**：重新排列明文字母顺序形成密文。
- **替换密码**：将明文中的每个字符或位替换为不同字符。
- **凯撒密码 (Caesar Cipher / ROT3)**：单表替换密码，将字母表每个字母向右移动N个位置（末尾折返）。
- **频率分析**：寻找密文中最常见字母并尝试替换，以破解简单密码的攻击方法。
- **多表替换密码**：同一消息中使用多个字母表，阻碍直接频率分析。
- **维吉尼亚密码 (Vigenère Cipher)**：著名多表替换密码，利用关键字和包含26次移位的单一图表转换。
- **周期分析**：基于密钥重复使用频率的二阶频率分析，多表替换密码易受此攻击。
- **一次性密码本**：不可破解的替换密码，为每个字母使用不同替换字母表，密钥必须随机生成、长度至少与消息等长、物理保护且仅使用一次。
- **流水密钥密码 / 书本密码**：从常见书籍/报纸中选择与消息等长的文本作为加密密钥，避免物理交换密码本。

> ⚠️ **考试陷阱：一次性密码本条件**
> 一次性密码本必须**同时满足**四个条件才不可破解：1) 密钥随机生成；2) 长度至少与消息等长；3) 物理保护；4) 仅使用一次。缺少任何一个即不再安全。

### 分组与流密码
| 类型 | 定义 |
|---|---|
| **分组密码** | 对消息的固定长度段（块）进行操作，将加密算法应用于整个消息块。 |
| **流密码** | 每次对消息或数据流的一个字符或位进行独立操作。 |

### 混淆与扩散
- **混淆**：使明文和密钥之间的关系极其复杂，让攻击者无法通过改变明文分析密文来确定密钥。
- **扩散**：明文中的一处更改会导致分布在整个密文中的多处更改。

> 💡 **记忆 aid：混淆 vs 扩散**
> - **混淆** 针对 **密钥**（关系复杂，找不到密钥）
> - **扩散** 针对 **明文**（一处改动，散布全身）

## 6.2 现代密码学

### 对称密钥算法
- **对称密钥算法 (私钥密码学)**：依赖分发共享秘密密钥，发送方和接收方用于加密和解密。主要用于批量加密，**仅提供机密性**。
- **临时密钥**：仅在单一会话中使用的对称密码学秘密密钥。
- **对称密钥可扩展性公式**：完全连接 n 个参与者所需的总密钥数为 **$n(n-1)/2$**。

> ⚠️ **考试陷阱：对称密钥数量计算**
> 考试常考 n 人通信需要多少对称密钥。代入公式 $n(n-1)/2$。例如10人需要 $10 \times 9 / 2 = 45$ 把密钥。缺乏可扩展性是对称加密的主要缺点。

### 非对称与混合密码学
- **非对称密钥算法**：每个用户分配公钥（共享）和私钥（保密）。相反且相关的密钥必须协同使用（公钥加密只能私钥解密，反之亦然）。
- **混合密码学**：结合两者优势，先用非对称密码学建立连接并交换对称密钥，随后使用速度更快的对称密码学进行会话。

### 哈希与完整性
- **消息摘要 / 哈希值**：由哈希算法产生的消息内容的固定长度摘要（指纹），结合数字签名实现完整性和不可否认性。
- **哈希碰撞**：哈希函数为两个不同消息产生相同摘要值，碰撞通常导致该哈希算法被弃用。

## 6.3 对称密码

### 分组密码操作模式
| 模式 | 全称 | 特点与机制 | 错误传播 |
|---|---|---|---|
| **ECB** | Electronic Codebook | 最简单最不安全；相同明文块产生相同密文块；无IV | 无 |
| **CBC** | Cipher Block Chaining | 加密前明文块与前一个密文块XOR；第一块用IV XOR | 当前块及下一块受影响 |
| **CFB** | Cipher Feedback | CBC的流密码版本；用内存缓冲区处理实时数据；用IV和链接 | 当前块及下一块受影响 |
| **OFB** | Output Feedback | 类似CFB，但明文与种子值XOR（非前一个密文块加密版） | 不传播 |
| **CTR** | Counter | 流密码；用递增计数器生成种子值；支持并行计算 | 不传播 |
| **GCM** | Galois/Counter | CTR模式 + 认证标签（数据真实性控制） | 不传播 |
| **CCM** | Counter with CBC-MAC | CTR模式(机密性) + CBC-MAC(真实性)；仅用于128位块长 | - |

> ⚠️ **考试陷阱：CBC模式的错误传播**
> 在CBC模式中，如果传输过程中一个密文块损坏，将导致**该数据块及下一个数据块**无法解密，但不会影响更后面的块。
> 
> ⚠️ **考试陷阱：ECB模式的缺陷**
> ECB模式下相同的明文块永远生成相同的密文块，因此绝不能用于加密超过一个块的数据（尤其是具有模式的数据，如位图），否则会被模式识别攻击。

> 💡 **记忆 aid：模式与错误传播**
> - 有"链接"机制的模式（CBC, CFB）会传播错误到下一块。
> - 使用"种子/计数器"的模式（OFB, CTR）不传播错误。
> - 带有"认证"的模式（GCM, CCM）同时提供机密性和完整性。

### 对称加密算法标准
| 算法 | 块大小 | 密钥长度/特征 | 状态 |
|---|---|---|---|
| **DES** | 64位 | 64位密钥（含8位奇偶校验，**有效56位**）；16轮XOR操作 | 已不安全 |
| **3DES** | 64位 | 使用相同DES算法但采用多个独立密钥 | 已被美联邦政府正式禁止 |
| **IDEA** | 64位 | 128位初始密钥，分解为52个16位子密钥；结合XOR和取模操作 | - |
| **Blowfish** | - | - | - |

> ⚠️ **考试陷阱：DES的密钥长度**
> DES密钥常被误认为是64位。实际上64位中包含8位奇偶校验位，**有效密钥长度仅为56位**。

### 对称密钥算法 (Symmetric Key Algorithms)

#### Blowfish
*   **定义**: 一种处理64位文本块的块密码。
*   **参数**: 允许使用32位到448位范围内的可变长度密钥。
*   **特性**: 在时间测试中被证实比IDEA和DES算法快得多。

#### SKIPJACK
*   **定义**: 由美国国家安全局（NSA）开发并批准用于托管加密标准（EES）的算法。
*   **参数**: 使用80位密钥操作64位文本块。
*   **特性**: 显著特点是支持加密密钥的托管。

#### Rivest 密码系列

| 算法 | 类型 | 块大小/流 | 密钥长度 | 关键特征 / 关联协议 |
| :--- | :--- | :--- | :--- | :--- |
| **RC4** | 流密码 | N/A (流) | 40位至2,048位 (可变) | 单轮加密。曾广泛集成于WEP、WPA、SSL和TLS，现因受多种攻击被认为**不安全**。 |
| **RC5** | 块密码 | 32、64或128位 (可变) | 0到2,040位 (可变) | 对较旧的RC2算法的改进。 |
| **RC6** | 块密码 | 128位 | 128、192或256位 | 作为RC5的下一版本开发，曾是AES的候选算法。 |

⚠️ **常见误区**: RC5和RC6是块密码，而RC4是流密码；RC5是对RC2的改进，**与RC4完全无关**。

#### 高级加密标准
*   **定义**: 由NIST选定取代DES的分组密码（基于Rijndael算法）。
*   **参数**: 专门处理128位的数据块。
*   **密钥与轮数映射**:

| 密钥长度 | 加密轮数 |
| :--- | :--- |
| 128位 | 10轮 |
| 192位 | 12轮 |
| 256位 | 14轮 |

🧠 **记忆辅助**: AES密钥轮数规律——"128起步10轮加，每加64位密钥，轮数加2" (10 -> 12 -> 14)。

#### CAST 系列算法

| 算法 | 网络结构 | 加密轮数 | 密钥长度 | 操作块大小 |
| :--- | :--- | :--- | :--- | :--- |
| **CAST-128** | Feistel网络 | 12或16轮 | 40到128位 | 64位 |
| **CAST-256** | Feistel网络 | 48轮 | 128、160、192、224或256位 | 128位 |

### 密钥管理与托管

#### 拆分知识原则
*   **定义**: 一种密钥存储的最佳实践，涉及将敏感密钥分为两半并分别提供给两个不同的人，他们必须协作才能重新创建完整的密钥。

#### 硬件安全模块
*   **定义**: 用于管理和保护组织加密密钥的专用企业级硬件设备。
*   **特性**: 比基于软件的存储方法更复杂且成本更高，但能提供额外的安全性。

#### 密钥托管方法

| 方法 | 定义 | 关键特征 |
| :--- | :--- | :--- |
| **公平密码系统** | 通信使用的秘密密钥被分成两部分或多部分并交给独立的第三方。 | 各部分单独无用；在政府获得合法授权时可重新组合以获取秘密密钥。 |
| **托管加密标准** | 为政府或其他授权代理提供解密密文的技术手段的密钥托管方法。 | 曾是**Clipper芯片**提议采用的方法（结合SKIPJACK算法）。 |

⚠️ **易混考点**: 拆分知识原则是**内部密钥存储**的最佳实践（双人控制）；公平密码系统和托管加密标准是**外部密钥托管**机制（涉及第三方/政府授权）。

### 6.4 密码生命周期

#### 密码生命周期
*   **核心概念**: 除一次性密码本外，所有密码系统都有其有限的生命周期。随着计算能力的提升，处理器最终将能够直接猜测出用于通信的加密密钥。

#### 摩尔定律
*   **定义**: 一种关于计算能力发展的趋势理论，指出最先进微处理器的处理能力大约每两年就会翻一番。

#### 算法和协议治理控制
*   **定义**: 为保护密码系统在所需保密期限内的完整性而实施的控制措施。
*   **具体措施**:
    1.  指定组织内可接受的密码算法。
    2.  根据信息的敏感度确定可接受的密钥长度。
    3.  明确允许使用的安全事务协议。
