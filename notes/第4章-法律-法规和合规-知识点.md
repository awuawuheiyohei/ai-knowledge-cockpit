# 第4章-法律-法规和合规-知识点

<!-- Refined section 1/1 -->

### 4.1 法律的分类

#### 刑法
*   **定义与作用**：维护社会和平与安全的法律基石。
*   **内容**：包含对谋杀、攻击、抢劫、纵火等行为的禁止。
*   **处罚范围**：社区服务、大小额罚款及监禁（剥夺公民自由）。

#### 民法
*   **定义与作用**：旨在维护社会秩序，规范非犯罪但需公正仲裁的个人与组织间事务。
*   **政府框架构建**：用于构建行政分支履行职责的政府框架，包括为政府活动提供预算、赋予行政分支制定行政法规的权力。

#### 行政法
*   **制定主体**：行政机构。
*   **表现形式**：行政命令、政策、程序、法规等。
*   **生效条件**：无需立法分支通过即可生效。
*   **约束限制**：
    *   必须符合现有民事及刑事法律。
    *   不得与立法机构的现行法律直接冲突。
    *   需遵守宪法。
    *   接受司法审查。

#### 组织安全合规的关键要求
*   需理解法律（行政法、民法、刑法等）的复杂性以应对监管环境。
*   重点关注法律、法规、调查及合规对组织安全的影响。
*   ⚠️ **必须寻求律师等专业人士帮助**以维持合法的安全。

### 4.2 法律

#### 信息安全专业人员的法律认知要点
*   需了解与信息技术相关的法律基础。
*   ⚠️ **核心陷阱**：关键不是精通法律，而是**知道何时咨询律师**。若处于法律“灰色区域”，应寻求专业法律建议。

#### NIST 信息安全程序的要素
*   **核心要素**：
    1.  定期风险评估
    2.  基于风险的策略程序
    3.  安全计划
    4.  安全意识培训
    5.  定期测试评估
    6.  补救行动流程
    7.  安全事件响应程序
    8.  业务连续性计划
*   **关键标准与框架**：

| 标准/框架 | 适用范围与说明 |
| :--- | :--- |
| **SP 800-53** | 联邦系统必用，行业基准 |
| **SP 800-171** | 非联邦系统保护受控非机密信息，联邦承包商常需遵守 |
| **CSF** (网络安全框架) | 自愿风险框架 |

#### 知识产权 (IP)
*   **定义**：企业的无形资产，需法律保护所有者权利。
*   **四种主要类型**：

| 类型 | 核心规则 | 记忆/避坑要点 |
| :--- | :--- | :--- |
| **版权** | 保护“原创作品作者”的**表达**，涵盖文学、音乐、软件（属文学作品）等8类。 | ⚠️ **不保护思想/流程**，只保护表达方式。 |
| **商标** | 区分一个产品/公司的颜色、声音、符号等，目的是避免市场混淆。 | 核心目的：**防混淆**。 |
| **专利** | 保护发明人独占权。要求：发明需**新、有用、非显而易见**。 | 记忆口诀：**新、有、非**（新有用且非显而易见）。 |
| **商业秘密** | 企业关键、披露会受损的信息。无需公开、无保护期限。 | ⚠️ 必须通过**访问控制、NDA、证明重视**来维持保护，否则可能丧失资格。 |

#### 软件与云服务许可协议

| 许可证类型 | 核心特征 |
| :--- | :--- |
| **永久许可证** | 一次性付费，无限期使用；支持和更新通常需额外付费。 |
| **订阅许可证** | 时间绑定，支付月度/年度等经常性费用；通常包含更新和支持。 |
| **开源许可证** | 通常可免费使用、修改和分发，但存在不同许可条件（如 GNU GPL、MIT License）。 |
| **免费软件** | 免费提供，但可能存在使用限制或缺少付费版本的功能。 |
| **企业许可协议 (ELAs)** | 供应商与大型组织间的全面协议，允许组织内全面部署，通常有优惠条款或折扣。 |
| **最终用户许可协议 (EULAs)** | 定义软件使用的权利与限制，通常在安装或初始设置时呈现。 |
| **云服务许可协议** | 针对互联网提供的 SaaS 的协议，注册在线服务时遇到，需确认条款后方可访问。 |

#### 进出口控制
*   **定义与目的**：各国实施的基于国家的规则和法律，管理哪些产品、技术和信息可以进出这些国家。旨在保护国家安全、个人隐私、经济福祉等。
*   **主要法规/协议**：

| 法规/协议 | 核心内容 |
| :--- | :--- |
| **ITAR** (国际武器贸易条例) | 美国法规，控制导弹、火箭、炸弹或《美国军火清单》等物品的出口。 |
| **EAR** (出口管理条例) | 覆盖商业用途但可能有军事应用的物品。涵盖物品列于《商业控制清单 (CCL)》（美国商务部维护），**包含信息安全产品类别**。 |
| **瓦塞纳尔协议** | 管理密码学带来的风险，同时促进贸易。允许特定国家交换/使用任何强度的密码系统，防止其落入恐怖分子手中。 |

#### 隐私法 (重点：欧盟 GDPR)
*   **GDPR (通用数据保护条例)**：
    *   **适用性**：适用于所有欧盟成员国的单一规则集。
    *   **监管机构 (SA)**：每个国家设立一个独立的监管机构，负责听取和调查投诉；数据主体有权向其提出投诉。
    *   **泄露报告**：⚠️ 隐私泄露必须在 **72 小时内**报告。
    *   **合法处理个人数据的七个原则**：
        1. 合法性、公平性和透明性
        2. 目的限制
        3. 数据最小化
        4. 准确性
        5. 存储限制
        6. 完整性和保密性
        7. 责任性

*(注：美国隐私法、加拿大PIPEDA、中国PIPL、南非POPIA及各州隐私法标注为中文考试不考察，已省略)*

### 4.4 合规

#### PCI DSS 4.0 概述和核心框架
*   **性质**：⚠️ **合同义务类合规要求（非法律强制）**。通过商家与交易处理银行的商户协议执行。
*   **管辖范围**：信用卡与借记卡信息安全。非商家但为商家存储、处理或传输信用卡信息的组织（如共享主机提供商）也需遵守。
*   **12项主要要求覆盖维度**：
    *   网络安全控制
    *   系统安全配置
    *   存储账户数据保护
    *   传输中持卡人数据强加密
    *   恶意软件防护
    *   系统软件安全开发维护
    *   按需知密的访问限制
    *   用户身份验证
    *   持卡人数据物理访问限制
    *   系统网络访问日志监控
    *   定期安全测试
    *   信息安全组织政策支持

#### 组织合规管理常见举措
*   配备全职 IT 合规人员。
*   职责：跟踪监管环境、监控控制以保障持续合规、协助合规审计、履行合规报告义务。

#### 合规审计类型
*   **内部/外部审计**：如财务审计中的 IT 控制审计以符合 SOX 法案。
*   **监管机构审计**：由监管机构或其代理执行。
*   **独立审计师验证**：⚠️ 部分法规（如 PCI DSS）要求聘请获批独立审计师验证控制，并**直接向监管机构提交报告**。

#### 合规报告要求
*   **内部报告**：向董事会审计委员会等定期报告。
*   **外部报告**：向外部利益相关者定期报告合规义务与状态。
*   **PCI DSS 特殊要求**：⚠️ 未进行第三方审计的组织需提交**自我评估报告**说明合规情况。

### 4.5 合同和采购

#### 供应商安全审查需求
*   **背景**：云服务及外部供应商存储、处理、传输敏感信息的使用增加。
*   **审查时机**：
    1.  初始供应商选择评估阶段。
    2.  持续的供应商治理审查过程。

#### 供应商治理审查的核心关注维度
*   敏感信息类型
*   信息保护控制
*   客户信息与其他客户的隔离机制
*   加密算法/密钥长度及密钥管理
*   供应商安全审计类型及客户访问权限
*   第三方依赖及合同安全条款延伸
*   数据存储/处理/传输位置及跨境影响
*   事件响应流程及客户通知机制
*   数据持续完整性与可用性保障
